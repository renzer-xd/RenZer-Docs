วิธีติดตั้ง OS Ubuntu Server

คู่มือแนะนำขั้นตอนการติดตั้งระบบปฏิบัติการ Ubuntu Server ตั้งแต่การเลือกเวอร์ชัน การตั้งค่าเครือข่าย โปรไฟล์ การตั้งค่าสิทธิ์ผู้แลระบบสูงสุด (Root) ไปจนถึงการเปิดใช้งาน SSH สำหรับเชื่อมต่อระยะไกล

1. การเลือกเวอร์ชันติดตั้ง​

  • เลือกเวอร์ชันที่คุณต้องการใช้งาน เช่น 20, 22, 24, หรือ 25
  • ข้อสำคัญ: ให้เลือกตัวเลือกการติดตั้งแบบ ไม่มีคำว่า Auto

2. ขั้นตอนเริ่มต้น​

  • เลือกเมนู Try or Install Ubuntu Server
  • หลังจากนั้นรอระบบทำการโหลดข้อมูลเริ่มต้นสักครู่

3. การตั้งค่าเครือข่าย (Network Configuration)​

การกำหนดค่าการตั้งค่า IP แบบคงที่ (Static IP):

  1. เลือกอินเทอร์เฟซการเชื่อมต่อ (ตัวอย่างเช่น ens33)
  2. เลือกเมนู Edit IPv4 -> ปรับรูปแบบเป็น Manual
  3. กำหนดค่ารายละเอียดเครือข่ายดังนี้:
    • Subnet: ใส่ IP ในคลาสที่เราได้มา โดยจำเป็นต้องให้ตัวสุดท้ายลงท้ายด้วย 0 ตามด้วยความกว้าง Subnet 24 หรือ 25
      • ตัวอย่าง: 103.224.86.0/24 หรือ 103.224.86.0/25
    • Address: ระบุ IP Address ที่เตรียมไว้ให้ลูกค้า
    • Gateway: ระบุเกตเวย์ (ตัวอย่างเช่น 103.224.86.1)
    • Name server: ระบุ DNS Server เช่น 8.8.8.8

4. การเลือกพื้นที่ติดตั้ง (Disk Configuration)​

  • เลือกดิสก์ (Disk) ที่ต้องการจัดทำการติดตั้งระบบปฏิบัติการ

5. การตั้งค่าผู้ใช้งาน (Profile Configuration)​

กำหนดบัญชีผู้ใช้งานระบบเริ่มต้น:

  • Your name: server
  • Your server's name: server
  • Pick a username: server
  • Choose a password: สยามดาราตัว123
  • Confirm your password: สยามดาราตัว123

6. การติดตั้ง SSH​

  • ติ๊กเลือก Install OpenSSH Server
  • รอให้ระบบดำเนินการติดตั้งข้อมูลทั้งหมดจนเสร็จสิ้น (Ready/Complete)

7. การตั้งค่าสิทธิ์ Root​

หลังจากติดตั้ง OS เสร็จเรียบร้อยแล้ว ให้ทำการกำหนดสิทธิ์ผู้ดูแลระบบสูงสุด (Root Privilege) ดังนี้:

  1. เข้าสู่ระบบ (Login) ด้วย Username และ Password ที่ตั้งค่าไว้ในขั้นตอนเริ่มต้น (ข้อ 5)
  2. รันคำสั่งต่อไปนี้เพื่อกำหนดรหัสผ่านใหม่ของ Root: 
    sudo passwd root
    หมายเหตุ: ระบบจะให้คุณกรอกรหัสผ่านเดิม (สยามดาราตัว123) ก่อน จากนั้นจึงกำหนดรหัสผ่านใหม่สำหรับบัญชี Root
  3. สลับการใช้งานเข้าสู่บัญชีผู้ใช้ Root ด้วยคำสั่ง: 
    su root

8. เปิดใช้งาน SSH และอนุญาตให้ Root ล็อกอินได้​

ปกติ Ubuntu Server จะไม่อนุญาตให้บัญชี Root เชื่อมต่อ SSH ได้โดยตรงเพื่อความปลอดภัย หากต้องการเปิดใช้งานให้ตั้งค่าดังนี้:

  1. เปิดไฟล์กำหนดค่าบริการ SSH ด้วยคำสั่ง: 
    nano /etc/ssh/sshd_config
  2. ตรวจสอบพอร์ตการใช้งาน: สังเกตพอร์ตแรกสุด โดยปกติจะเป็น Port 22
  3. ค้นหาหัวข้อ PermitRootLogin:
    • ให้ลบเครื่องหมาย # ที่อยู่ด้านหน้าออก (Uncomment)
    • แก้ไขคำว่า prohibit-password ให้เป็น yes
    • บันทึกและออกจากไฟล์โดยกดปุ่ม Ctrl + X จากนั้นกด y และกด Enter

9. การเปลี่ยนรหัสผ่านเพื่อความปลอดภัยก่อนส่งงาน​

เมื่อกำหนดสิทธิ์เสร็จสิ้น ให้ทำการเปลี่ยนรหัสผ่านของ Root เป็นรหัสที่มีความปลอดภัยสูง:

  1. รันคำสั่ง: 
    passwd root
  2. ทำการกำหนดรหัสผ่านแบบสุ่มที่มีส่วนผสมของ อักษรตัวพิมพ์ใหญ่ / อักษรพิเศษ / อักษรตัวพิมพ์เล็ก / ตัวเลข
  3. ข้อปฏิบัติสำคัญ: ให้ทำการเปลี่ยนรหัสผ่านนี้ก่อนส่งมอบงานให้ลูกค้า และแจ้งรหัสผ่านใหม่นี้ให้แก่ลูกค้า

10. ทดสอบการเชื่อมต่อ SSH​

  • ทดลองเชื่อมต่อ SSH เข้ามายังเซิร์ฟเวอร์ด้วยโปรแกรมรีโมทต่างๆ เพื่อยืนยันว่าสามารถล็อกอินเข้าเซิร์ฟเวอร์ได้ปกติ

11. การออกจากสิทธิ์ Root​

  • เมื่อทำงานหรือตรวจสอบเสร็จแล้ว หากต้องการออกจากสิทธิ์ Root ให้ใช้คำสั่ง: 
    exit

12. ข้อกำหนดการเข้าใช้หลังจากนี้​

  • ตั้งแต่นี้เป็นต้นไป ให้ใช้บัญชี root ในการเข้าใช้งานระบบทั้งหมด รวมถึงเวลาส่งงานลูกค้าให้แจ้งข้อมูลการเข้าสู่ระบบด้วยสิทธิ์ root เท่านั้น

13. ตรวจสอบการตั้งค่าเครือข่ายอินเทอร์เน็ตเพิ่มเติม​

ก่อนการทดสอบเชื่อมต่อ SSH สามารถเปิดตรวจสอบข้อมูลการตั้งค่าเครือข่าย (เน็ตการ์ด) ของคุณได้ผ่านคำสั่ง Netplan:

nano /etc/netplan/50-cloud-init.yaml

คำสั่งนี้ช่วยตรวจสอบว่า IP Address, Gateway, และ DNS ที่ได้ระบุไว้ในขั้นตอนติดตั้งได้รับการกำหนดค่าลงในเน็ตการ์ดอย่างถูกต้องหรือไม่

14. สิ่งที่ควรทำหลังติดตั้งระบบเสร็จสิ้น (Post-Installation Checklist)​

หลังจากเปิดใช้งาน SSH และเข้าสู่ระบบเรียบร้อยแล้ว แนะนำให้ดำเนินการตามขั้นตอนต่อไปนี้ทันทีเพื่อความปลอดภัย เสถียรภาพ และความเร็วของเซิร์ฟเวอร์ก่อนเริ่มนำไปใช้งานจริงหรือส่งมอบงานให้ลูกค้า:

1. อัปเดตแพตช์ความปลอดภัยและฐานข้อมูลระบบ (Update & Upgrade Packages)​

อัปเดตรายชื่อแพ็กเกจซอฟต์แวร์ทั้งหมดในคลัง (Repository) และทำการอัปเกรดซอฟต์แวร์รวมถึงอุดช่องโหว่ความปลอดภัยเวอร์ชันล่าสุด:

sudo apt update && sudo apt upgrade -y
tip

ทำไมควรทำทันที? ระบบปฏิบัติการในแผ่นอิมเมจติดตั้งมักจะเป็นเวอร์ชันเก่าจากวันที่สร้างแผ่น การอัปเดตทันทีจะช่วยป้องกันการโดนโจมตีผ่านบักเดิมของซอฟต์แวร์ระบบได้

2. ตั้งค่าโซนเวลาของเซิร์ฟเวอร์ (Set System Timezone)​

โดยปกติ Ubuntu Server จะตั้งค่าเริ่มต้นเวลาเป็นโซน UTC เสมอ แนะนำให้แก้ไขเวลาให้ตรงกับประเทศไทย (GMT+7) เพื่อความสะดวกในการอ่าน Log และตั้งเวลารันงานอัตโนมัติ (Cron job):

sudo timedatectl set-timezone Asia/Bangkok
  • คำสั่งตรวจสอบสถานะเวลาปัจจุบัน: 
    timedatectl
    (สังเกตบรรทัด Time zone: Asia/Bangkok (EST, +0700) และเวลาในบรรทัด Local time จะต้องตรงกับเวลาประเทศไทย ณ ปัจจุบัน)

3. ติดตั้งชุดเครื่องมือพื้นฐานที่จำเป็น (Install Essential Utilities)​

ติดตั้งแพ็กเกจพื้นฐานที่ระบบจำเป็นต้องเรียกใช้งานบ่อยในการบริหารจัดการไฟล์ ถอนสคริปต์ ดึงข้อมูล และทดสอบเน็ตเวิร์ก:

sudo apt install -y curl wget git unzip tar build-essential software-properties-common dnsutils
  • คำย่อและหน้าที่สำคัญ:
    • curl / wget : ใช้ดาวน์โหลดไฟล์และเรียกยิงทดสอบ API
    • git : ใช้โคลน source code หรือดึงข้อมูลโปรเจกต์
    • unzip / tar : ใช้จัดการบีบอัดและแตกไฟล์ประเภท zip / tar.gz
    • dnsutils : ติดตั้งชุดคำสั่งทดสอบ DNS เช่น nslookup หรือ dig

4. เปิดใช้งานระบบ TCP BBR เพื่อเร่งความเร็วเครือข่าย (Enable Google BBR)​

BBR (Bottleneck Bandwidth and RTT) เป็นเทคโนโลยีควบคุมการจราจรเครือข่ายของ Google ที่ช่วยจัดการลำดับส่งข้อมูลผ่าน TCP ให้ฉลาดขึ้น ช่วยลดปัญหาปิงสูง (Latency) และเพิ่มความเร็วของการรับส่งข้อมูลบนเซิร์ฟเวอร์ได้อย่างมหาศาล (โดยเฉพาะงาน Web Server หรือการดึงส่งไฟล์ขนาดใหญ่)

  • วิธีการเปิดใช้งาน:
    1. เพิ่มกฎการควบคุม TCP เข้าไปที่ไฟล์คอนฟิกระบบหลัก /etc/sysctl.conf ด้วยคำสั่ง: 
      echo "net.core.default_qdisc=fq" | sudo tee -a /etc/sysctl.conf
      echo "net.ipv4.tcp_congestion_control=bbr" | sudo tee -a /etc/sysctl.conf
    2. รันคำสั่งบังคับให้ระบบโหลดค่าตั้งค่าใหม่เข้ามาทำงานทันที: 
      sudo sysctl -p
    3. ตรวจสอบสถานะยืนยันการเปิดใช้งานสำเร็จ: 
      sysctl net.ipv4.tcp_congestion_control
      (ผลลัพธ์จะต้องตอบกลับมาว่า net.ipv4.tcp_congestion_control = bbr จึงจะถือว่าเปิดใช้งานสำเร็จ)

5. ทำความสะอาดขยะระบบหลังการตั้งค่า (System Clean Up)​

เคลียร์และลบไฟล์ขยะจากการติดตั้งหรืออัปเกรดเวอร์ชันซอฟต์แวร์ย้อนหลัง เพื่อช่วยคืนพื้นที่จัดเก็บข้อมูลให้ฮาร์ดดิสก์:

sudo apt autoremove -y && sudo apt clean
  • คำอธิบาย:
    • autoremove : ลบไฟล์ Library พ่วงเก่าๆ ที่ไม่มีเซอร์วิสใดต้องการเรียกใช้งานแล้ว
    • clean : เคลียร์ไฟล์ Cache ซอฟต์แวร์ดาวน์โหลด (.deb) ที่คงค้างอยู่ในเครื่องหลังติดตั้งเสร็จสิ้น
