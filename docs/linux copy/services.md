# คู่มือการควบคุมระบบและบริการ (Services) อย่างละเอียด

ทำความเข้าใจระบบผู้ควบคุมการทำงานเบื้องหลังของลีนุกซ์ (init system) ของระบบปฏิบัติการผ่านเครื่องมือ **systemd** และการวิเคราะห์ปัญหาระบบด้วย **journalctl**

---

## ⚙️ ทำความรู้จักกับ systemd และ Services

ระบบปฏิบัติการ Linux สมัยใหม่ส่วนใหญ่ (รวมถึง Ubuntu, Debian, CentOS, RedHat) ใช้ **systemd** เป็นเครื่องมือหลักในการเริ่มต้นระบบ ควบคุมโปรเซส และจัดการบริการเบื้องหลัง (System Services หรือที่เรียกว่า Daemon)

รายการคำสั่งควบคุมทั้งหมดจะทำงานผ่านแอปพลิเคชันตัวสั่งการหลักที่ชื่อว่า **`systemctl`** และสืบค้นล็อกข้อมูลผ่าน **`journalctl`**

---

## 🛠️ คู่มือคำสั่งจัดการบริการด้วย `systemctl` (System Control)

โครงสร้างคำสั่งทั่วไปในการควบคุมมีลักษณะดังนี้:
```bash
systemctl <คำสั่งปฏิบัติงาน> <ชื่อเซอร์วิส>
```

### 1. การควบคุมสถานะการทำงานทั่วไป (Core Controls)

* **`systemctl status`**: ใช้ตรวจสอบสถานะปัจจุบันของบริการ (กำลังรันอยู่, หยุดทำงาน, หรือเกิดปัญหาผิดพลาด)
  ```bash
  systemctl status sshd
  ```
  *ข้อมูลในสเตตัสที่จะพบ:*
  * `active (running)` : เซอร์วิสกำลังทำงานอยู่ปกติ
  * `inactive (dead)` : เซอร์วิสปิดอยู่
  * `failed` : เกิดความเสียหาย ทำให้ไม่สามารถรันระบบขึ้นมาได้

* **`systemctl start`**: สั่งเริ่มเปิดการทำงานของบริการนั้นๆ ทันที
  ```bash
  systemctl start nginx
  ```

* **`systemctl stop`**: สั่งหยุดและปิดการทำงานของบริการนั้นๆ ทันที
  ```bash
  systemctl stop nginx
  ```

* **`systemctl restart`**: สั่งให้หยุดบริการและเริ่มต้นทำงานใหม่ทันที (นิยมใช้สำหรับรีเซ็ตหลังจากมีการแก้ไขไฟล์การตั้งค่า Config ของระบบ)
  ```bash
  systemctl restart nginx
  ```

* **`systemctl reload`**: สั่งให้อ่านไฟล์คอนฟิกใหม่โดยไม่มีการปิดและเริ่มบริการใหม่ (ดีกว่า restart เนื่องจากช่วยป้องกันบริการล่มชั่วคราวขณะผู้ใช้กำลังใช้เว็บอยู่)
  ```bash
  systemctl reload nginx
  ```

---

### 2. การกำหนดค่าอัตโนมัติเมื่อเปิดเครื่อง (Startup Configuration)

* **`systemctl enable`**: ตั้งค่าให้บริการเริ่มเปิดการทำงานของตนเองโดยอัตโนมัติทุกครั้งที่มีการเปิดเครื่องเซิร์ฟเวอร์ขึ้นมาใหม่ (Startup)
  ```bash
  systemctl enable nginx
  ```

* **`systemctl disable`**: ยกเลิกการตั้งค่าเริ่มการทำงานอัตโนมัติเมื่อเปิดเครื่อง (หากรีสตาร์ตเครื่อง เซอร์วิสนี้จะปิดอยู่จนกว่าเราจะมากดเริ่มทำงานด้วยตัวเอง)
  ```bash
  systemctl disable nginx
  ```

---

### 3. คำสั่งย่อยเพื่อตรวจสอบและสืบค้นเชิงลึก (Helper Commands)

* **`systemctl is-active`**: ตรวจสอบอย่างย่อว่าเซอร์วิสกำลังเปิดทำงานอยู่หรือไม่ (ผลลัพธ์จะแจ้งเพียงคำว่า `active` หรือ `inactive`)
  ```bash
  systemctl is-active ssh
  ```

* **`systemctl is-enabled`**: ตรวจสอบอย่างย่อว่ามีการตั้งค่าให้เริ่มต้นอัตโนมัติเมื่อบูตเซิร์ฟเวอร์ไว้หรือไม่ (ผลลัพธ์จะขึ้นคำว่า `enabled` หรือ `disabled`)
  ```bash
  systemctl is-enabled ssh
  ```

* **`systemctl list-units`**: แสดงรายชื่อบริการและยูนิตระบบทั้งหมดที่กำลังทำงานอยู่ในขณะนั้น
  ```bash
  systemctl list-units --type=service      # แสดงรายชื่อของ Services ทั้งหมดในระบบ
  ```

---

## 📋 คู่มือสืบค้น Log และตรวจหาข้อผิดพลาดด้วย `journalctl`

เมื่อระบบบริการเกิดปัญหาและขึ้นสถานะเป็น `failed` หน้าจอเทอร์มินัลมักจะไม่แจ้งสาเหตุตรงๆ เครื่องมือ **`journalctl`** จะทำหน้าที่เข้าไปขุดค้นไฟล์ Log ระบบของ systemd เพื่อหาสาเหตุที่ทำให้ระบบหยุดทำงาน

### คำสั่งสำคัญในการขุดค้น Log:

* **`journalctl -u` (Unit log):** เปิดแสดง Log การทำงานทั้งหมดเฉพาะเจาะจงของบริการที่ระบุ
  ```bash
  journalctl -u nginx
  ```

* **`journalctl -n` (Number of lines):** แสดงเฉพาะแถว Log ส่วนท้ายสุดตามจำนวนที่ระบุ (ช่วยให้อ่านง่ายขึ้น ไม่ต้องดูข้อมูลเก่าเก็บยาวเกินไป)
  ```bash
  journalctl -n 50 -u nginx                # แสดง Log 50 บรรทัดล่าสุดของ Nginx
  ```

* **`journalctl -f` (Follow):** การดู Log แบบเกาะติดความเคลื่อนไหวสด (คล้าย `tail -f`) ระบบจะเลื่อนและพิมพ์แถว Log ใหม่ขึ้นมาให้ดูทันทีเมื่อเกิดเหตุการณ์
  ```bash
  journalctl -f -u sshd                    # เฝ้าดูการทำงานการพยายามล็อกอิน SSH แบบเรียลไทม์
  ```
  > [!TIP]
  > **การออกจากหน้าระเบียง Log:**
  > หากต้องการหยุดการติดตามหรือออกจากหน้ารายงานของ journalctl ให้กดปุ่ม **`q`** บนคีย์บอร์ด (หรือกด **`Ctrl + C`** กรณีที่รันด้วยโหมดเกาะติด `-f`)

* **`journalctl -xe` (Extended info / tail e):** คำสั่งกู้ภัยเมื่อบริการมีปัญหาล่มตอน Start หรือ Restart คำสั่งนี้จะพากระโดดไปยังช่วงท้ายสุดของ Log ระบบ และแสดงคำอธิบายเชิงวิเคราะห์ความผิดพลาดระดับลึก
  ```bash
  journalctl -xe
  ```

* **`journalctl --since` (Filter by time):** กรองดูเฉพาะความเคลื่อนไหวตามกรอบเวลาที่ระบุ
  ```bash
  journalctl --since "1 hour ago"          # ดูประวัติเหตุการณ์ที่เกิดในช่วง 1 ชั่วโมงที่ผ่านมา
  journalctl --since "yesterday"           # ดูเหตุการณ์ตั้งแต่เมื่อวานนี้เป็นต้นมา
  ```

## 🔄 การปิดระบบและรีสตาร์ตเครื่องเซิร์ฟเวอร์ (Shutdown & Reboot Controls)

การปิดเครื่อง (Shutdown) หรือการรีสตาร์ตระบบ (Reboot) บน Linux จำเป็นต้องใช้สิทธิ์ผู้ดูแลระบบสูงสุด (`sudo` หรือ `root`) เพื่อให้มั่นใจว่าระบบจะบันทึกสถานะและปิดบริการต่างๆ ลงอย่างปลอดภัย:

### A. คำสั่งปิดระบบ: `shutdown`
* **`sudo shutdown now`**: สั่งปิดเซิร์ฟเวอร์ลงทันทีและตัดกระแสไฟ (เป็นคำสั่งที่นิยมใช้ที่สุดก่อนถอดสายไฟ)
  ```bash
  sudo shutdown now
  ```
* **`sudo shutdown +15`**: ตั้งเวลาปิดเครื่องเซิร์ฟเวอร์ล่วงหน้าอีก 15 นาที (ผู้ใช้ที่เชื่อมต่อ SSH อยู่จะได้รับข้อความแจ้งเตือนนับถอยหลัง)
  ```bash
  sudo shutdown +15 "Server will shutdown for hardware upgrade"
  ```
* **`sudo shutdown 22:00`**: ตั้งกำหนดการให้เซิร์ฟเวอร์ปิดตัวลงเวลา 22:00 น.
  ```bash
  sudo shutdown 22:00
  ```
* **`sudo shutdown -c`**: สั่ง **ยกเลิก (Cancel)** กฎการนับถอยหลังการปิดเครื่องที่ตั้งไว้ก่อนหน้านี้
  ```bash
  sudo shutdown -c
  ```
* **`sudo shutdown -r now`**: สั่งให้ระบบปิดเครื่องแล้วทำการ **เริ่มต้นระบบใหม่ (Reboot) ทันที** (ทำงานเช่นเดียวกับคำสั่ง `reboot`)
  ```bash
  sudo shutdown -r now
  ```

---

### B. คำสั่งเริ่มต้นระบบใหม่: `reboot`
* **`sudo reboot`**: สั่งปิดเซอร์วิสทั้งหมดและรีสตาร์ตเครื่องเซิร์ฟเวอร์ทันทีอย่างปลอดภัย
  ```bash
  sudo reboot
  ```
* **`sudo reboot -f` (Force):** บังคับให้ระบบทำการบูตเครื่องใหม่ทันทีโดยไม่ขอเจรจาหรือรอปิดโปรแกรมที่ค้างอยู่ (อาจทำให้ข้อมูลที่ไม่ได้เซฟสูญหายหรือไฟล์ระบบเสียหายได้ ควรใช้กรณีจำเป็นเท่านั้น)
  ```bash
  sudo reboot -f
  ```

---

### C. คำสั่งสั่งปิดเครื่องแบบอื่น: `poweroff` & `halt`
* **`sudo poweroff`**: สั่งหยุดการทำงานของระบบปฏิบัติการและส่งสัญญาณ ACPI ไปตัดไฟที่บอร์ดจ่ายไฟทันที
  ```bash
  sudo poweroff
  ```
* **`sudo halt`**: สั่งหยุดการทำงานของหน่วยประมวลผล (CPU) และหน่วยความจำทั้งหมด แต่ระบบปฏิบัติการบางรุ่นอาจจะไม่สั่งตัดกำลังไฟที่ฮาร์ดแวร์
  ```bash
  sudo halt
  ```

---

### D. ทางเลือกยุคใหม่ผ่าน `systemctl` (systemd)
ในระบบ Linux สมัยใหม่ ทุกคำสั่งปิด/รีสตาร์ตระบบด้านบน จะถูกแปลงไปเรียกใช้งานคำสั่งของ systemd เบื้องหลังดังนี้:
* **`sudo systemctl reboot`**: สั่งรีสตาร์ตระบบผ่าน systemd
  ```bash
  sudo systemctl reboot
  ```
* **`sudo systemctl poweroff`**: สั่งปิดเครื่องและตัดไฟผ่าน systemd
  ```bash
  sudo systemctl poweroff
  ```
