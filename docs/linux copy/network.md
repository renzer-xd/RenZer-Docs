# คู่มือการตั้งค่าและตรวจเช็คเครือข่าย (Network) อย่างละเอียด

ทำความเข้าใจสถาปัตยกรรมเครือข่าย ตรวจสอบหมายเลข IP ค้นหาพอร์ตการเชื่อมต่อ ดาวน์โหลดไฟล์ผ่านระบบคำสั่ง ตรวจสอบเส้นทางเครือข่าย และป้องกันระบบด้วยไฟร์วอลล์ (Firewall) บนระบบปฏิบัติการ Linux

---

## 🌐 1. การตรวจสอบอินเทอร์เฟซเครือข่ายและหมายเลข IP (IP & Interface Information)

เมื่อคุณเชื่อมต่อกับระบบเซิร์ฟเวอร์ คำสั่งแรกที่ใช้สำหรับตรวจสอบช่องทางเชื่อมต่อและหมายเลข IP ของเน็ตการ์ด (Network Interface Card) ได้แก่:

### A. คำสั่ง `ip a` (ip address)
* **คำอธิบาย:** เป็นเครื่องมือมาตรฐานสมัยใหม่ที่ใช้แสดงและจัดการเส้นทาง (Routing) รวมถึงหมายเลข IP Address บนเน็ตการ์ดทุกตัวในเครื่อง
* **ตัวอย่างการรัน:**
  ```bash
  ip a
  # หรือพิมพ์แบบเต็ม
  ip address show
  ```
* **รายละเอียดผลลัพธ์ที่สำคัญ:**
  * **`lo` (Loopback):** การ์ดเครือข่ายเสมือนภายในเครื่อง (ปกติจะมี IP เป็น `127.0.0.1` เสมอ ใช้สำหรับรันเซอร์วิสให้ระบบคุยกันเองภายใน)
  * **`eth0` / `ens33` / `enp0s3`:** ชื่อเน็ตการ์ดจริงที่เชื่อมต่อกับระบบแลนหรืออินเทอร์เน็ตภายนอก (ชื่อจะต่างกันไปตามชนิดจำลองหรือฮาร์ดแวร์จริง)
  * **`inet`:** บรรทัดที่ระบุหมายเลข IPv4 ของเน็ตการ์ดตัวนั้น เช่น `inet 103.224.86.15/24`
  * **`link/ether`:** แสดงหมายเลขประจำฮาร์ดแวร์เน็ตการ์ด (MAC Address)
  * **`state UP`:** บอกสถานะเน็ตการ์ดว่ากำลังเปิดทำงานพร้อมใช้งานอยู่

### B. คำสั่ง `ifconfig` (Interface Configuration)
* **คำอธิบาย:** เครื่องมือรุ่นดั้งเดิม (Legacy tool) ที่แสดงข้อมูลเน็ตการ์ดคล้ายกับ `ip a` แต่อ่านข้อมูลได้ง่ายกว่า
* **ตัวอย่างการรัน:**
  ```bash
  ifconfig
  ```
  > [!NOTE]
  > **กรณีไม่พบคำสั่ง `ifconfig`:**
  > ใน Linux เวอร์ชันใหม่ๆ (เช่น Ubuntu 20.04 ขึ้นไป) คำสั่งนี้อาจไม่ได้ติดตั้งมาให้ตั้งแต่เริ่มแรก หากต้องการใช้งาน คุณต้องทำการดาวน์โหลดชุดเครื่องมือ `net-tools` เพิ่มเติมผ่านคำสั่ง:
  > ```bash
  > sudo apt update && sudo apt install net-tools
  > ```

## 🔌 2. การตรวจสอบการเชื่อมต่อและพอร์ตที่กำลังเปิด (Sockets & Ports Monitoring)

การเปิดบริการระบบ เช่น Web Server (พอร์ต 80/443), SSH Server (พอร์ต 22), หรือ Database Server (พอร์ต 3306/5432) เซอร์วิสเหล่านั้นจะรอเปิดการเชื่อมต่อผ่านพอร์ตที่ระบุ หากมีปัญหาระบบเข้าใช้งานไม่ได้ คุณสามารถตรวจสอบการเปิดทำงานของพอร์ตเหล่านี้ได้ผ่านคำสั่ง:

### A. คำสั่ง `ss` (Socket Statistics)
* **คำอธิบาย:** เครื่องมือตรวจสอบซ็อกเก็ตเครือข่ายยุคใหม่ที่ทำงานได้เร็วและละเอียดมากที่สุด มักใช้แทนที่ `netstat` ที่ล่าสมัยไปแล้ว
* **พารามิเตอร์ออปชันยอดนิยม:**
  * `-t` : ค้นหาเฉพาะการเชื่อมต่อโปรโตคอล TCP (TCP Sockets)
  * `-u` : ค้นหาเฉพาะการเชื่อมต่อโปรโตคอล UDP (UDP Sockets)
  * `-l` : แสดงเฉพาะพอร์ตที่กำลังเปิดรอรับการเชื่อมต่ออยู่ (Listening Sockets)
  * `-p` : แสดงชื่อของโปรเซสและ PID ที่เป็นเจ้าของเปิดพอร์ตนั้นๆ (ต้องใช้ร่วมกับสิทธิ์ `sudo`)
  * `-n` : แสดงหมายเลขพอร์ตและ IP เป็นตัวเลขดิบโดยตรง ไม่ต้องเปลี่ยนตัวเลขพอร์ตเป็นชื่อบริการ (เช่น แสดง `22` แทนคำว่า `ssh`)
* **ตัวอย่างการรัน:**
  ```bash
  # ตรวจดูพอร์ต TCP/UDP ทั้งหมดที่กำลังเปิดรับการเชื่อมต่อพร้อมชื่อโปรเซสผู้รัน (นิยมใช้บ่อยที่สุด)
  sudo ss -tulpn
  ```
* **ตัวอย่างการตีความหมายคอลัมน์:**
  * `State: LISTEN` : พอร์ตกำลังสแตนด์บายรอรับข้อมูล
  * `Local Address:Port: 0.0.0.0:80` : เปิดบริการให้เครื่องภายนอกเข้าพอร์ต 80 ได้จากทุก IP ทั่วโลก
  * `Local Address:Port: 127.0.0.1:3306` : เปิดบริการฐานข้อมูลให้เข้าใช้ได้เฉพาะภายในเซิร์ฟเวอร์ตัวเองเท่านั้น (IP ภายนอกหมดสิทธิ์เชื่อมต่อโดยตรง)
  * `Process: users:(("nginx",pid=1120,fd=6))` : เซอร์วิส Nginx (PID 1120) เป็นผู้เปิดพอร์ตนี้

### B. คำสั่ง `netstat` (Network Statistics)
* **คำอธิบาย:** เครื่องมือตรวจสอบพอร์ตยอดนิยมดั้งเดิม ถึงแม้จะเก่าแต่ยังมีคนใช้เยอะมาก
* **ตัวอย่างการรัน:**
  ```bash
  # ตรวจดูพอร์ตทั้งหมดที่กำลังเปิดรอเชื่อมต่อในรูปแบบตัวเลข
  sudo netstat -plntu
  ```
  *(หากระบบเตือนว่าไม่พบคำสั่ง สามารถติดตั้งได้ผ่านแพ็กเกจ `net-tools` เช่นเดียวกับ `ifconfig`)*

## 📡 3. การทดสอบและค้นหาทางเครือข่าย (Network Testing & Diagnostics)

### A. คำสั่ง `ping` (Packet InterNet Groper)
* **คำอธิบาย:** ใช้ส่งแพ็กเกจข้อมูลขนาดเล็ก (ICMP Echo Request) ไปยังเซิร์ฟเวอร์ปลายทาง เพื่อทดสอบการตอบสนองว่าเซิร์ฟเวอร์ปลายทางเปิดอยู่หรือไม่ และประเมินคุณภาพความเสถียรของสัญญาณเครือข่าย
* **ออปชันย่อยที่สำคัญ:**
  * `-c <จำนวน>` : กำหนดจำนวนครั้งในการส่งสัญญาณ Ping แล้วหยุดทำงานโดยอัตโนมัติ (ปกติบน Linux หากไม่ระบุ ระบบจะ Ping ต่อเนื่องไปเรื่อยๆ ไม่มีวันหยุด)
* **ตัวอย่างการรัน:**
  ```bash
  # ทดสอบการเชื่อมต่อไปยัง IP ปลายทางแบบต่อเนื่อง
  ping 103.224.86.1
  
  # ทดสอบส่งสัญญาณ Ping ไปที่ google.com จำนวน 4 ครั้งแล้วหยุดทำงาน
  ping -c 4 google.com
  ```
* **รายละเอียดการอ่านผลลัพธ์:**
  * `time=12.5 ms` : ค่าความหน่วงเวลาในการส่งรับข้อมูล (Latency) ยิ่งน้อยยิ่งตอบสนองได้เร็ว
  * `packet loss` : อัตราการทำข้อมูลสูญหายระหว่างทาง หากเป็น `0% packet loss` แปลว่าเครือข่ายเสถียรดีมาก หากมีเปอร์เซ็นต์หลุดหาย แสดงว่าสัญญาณอินเทอร์เน็ตมีปัญหา
  * **การยกเลิกหน้าจอ Ping:** หาก Ping ค้างอยู่ ให้กดปุ่ม **`Ctrl + C`** เพื่อสั่งยกเลิกการทำงานและสรุปผล

### B. คำสั่ง `traceroute`
* **คำอธิบาย:** แสดงเส้นทางและอุปกรณ์เน็ตเวิร์ก (Router/Gateway/Hops) แต่ละชิ้นที่แพ็กเกจข้อมูลวิ่งผ่านจากเครื่องเราไปจนถึงเครื่องปลายทาง ช่วยจุดประกายไอเดียการไล่หาจุดที่สัญญาณอินเทอร์เน็ตขาดหายหรือคอขวด
* **ตัวอย่างการรัน:**
  ```bash
  traceroute google.com
  ```
  > [!NOTE]
  > **การติดตั้งเครื่องมือเพิ่มเติม:**
  > หากพิมพ์แล้วไม่พบคำสั่งในเครื่อง สามารถลงแพ็กเกจเพิ่มเติมได้ผ่าน:
  > ```bash
  > sudo apt update && sudo apt install traceroute
  > ```

## 📥 4. คำสั่งดาวน์โหลดและร้องขอเว็บผ่านเทอร์มินัล (Data Downloader & HTTP client)

การดาวน์โหลดข้อมูลหรือเรียกดู API ผ่าน Command Line มี 2 คำสั่งที่ถูกนำไปประยุกต์ใช้บ่อยที่สุด:

### A. คำสั่ง `wget` (Web Get)
* **คำอธิบาย:** ออกแบบมาเพื่อทำหน้าที่ดาวน์โหลดไฟล์ต่างๆ จากอินเทอร์เน็ตโดยตรงผ่านโปรโตคอล HTTP, HTTPS, FTP
* **ออปชันย่อยที่สำคัญ:**
  * `-O <ชื่อไฟล์ปลายทาง>` : ดาวน์โหลดไฟล์ลงมาเก็บพร้อมทำการเปลี่ยนชื่อตามที่เรากำหนด
  * `-c` : บังคับการดาวน์โหลดไฟล์ต่อจากจุดเดิมที่เคยดาวน์โหลดค้างไว้หากเน็ตตัดระหว่างทาง (Resume)
* **ตัวอย่างการรัน:**
  ```bash
  # ดาวน์โหลดไฟล์โดยตรงและเก็บชื่อไฟล์ตามต้นฉบับบนอินเทอร์เน็ต
  wget https://wordpress.org/latest.tar.gz
  
  # ดาวน์โหลดไฟล์พร้อมระบุชื่อใหม่และบันทึกไว้ในพาธอื่น
  wget -O /tmp/wp.tar.gz https://wordpress.org/latest.tar.gz
  ```

### B. คำสั่ง `curl` (Client URL)
* **คำอธิบาย:** เครื่องมือรับส่งข้อมูลผ่าน URL ที่มีความยืดหยุ่นสูงมาก รองรับการทำงานสารพัดโปรโตคอล นิยมใช้ส่งชุด API Request (GET, POST, PUT, DELETE) ตรวจสอบ Response Header หรือโหลดข้อมูล
* **ออปชันย่อยที่สำคัญ:**
  * `-O` (ตัวใหญ่) : ดาวน์โหลดไฟล์มาเก็บไว้โดยใช้ชื่อไฟล์ต้นทางจาก URL
  * `-o` (ตัวเล็ก) : ดาวน์โหลดไฟล์มาเก็บโดยตั้งชื่อและระบุตำแหน่งพาธใหม่
  * `-I` : ดึงเฉพาะส่วนหัวข้อมูลตอบรับของเว็บเซิร์ฟเวอร์ (HTTP Headers) มาตรวจสอบโดยไม่ต้องดึงตัวเนื้อหาเว็บทั้งหมด (ดีมากสำหรับการเช็คประเภท Web Server และ HTTP Status Code)
  * `-X <METHOD>` : กำหนดประเภทของการทำงาน HTTP Request เช่น POST, PUT, DELETE (ค่าเริ่มต้นจะเป็น GET)
  * `-H <HEADER_STRING>` : การแนบข้อมูลหัวข้อพิเศษ (Header Parameter) เช่น API Key หรือการกำหนด Content-Type
  * `-d <DATA>` : การแนบชุดข้อมูล payload/body ส่งไปกับคำสั่ง (มักใช้ควบคู่กับ POST)
* **ตัวอย่างการรัน:**
  ```bash
  # ตรวจเช็คข้อมูล Response Header ของเว็บไซต์ปลายทาง
  curl -I https://google.com
  
  # ดาวน์โหลดรูปภาพมาเก็บไว้ในเครื่อง
  curl -O https://example.com/images/logo.png
  
  # ส่งคำขอแบบ POST แนบชุดข้อมูล JSON ไปยัง API Server ปลายทาง
  curl -X POST -H "Content-Type: application/json" -d '{"username":"admin","pass":"1234"}' https://api.mysite.com/login
  ```

## 🛡️ 5. การรักษาความปลอดภัยด้วยไฟร์วอลล์ระบบ `ufw` (Uncomplicated Firewall)

**UFW** เป็นเครื่องมือตั้งค่าจัดการไฟร์วอลล์ยอดนิยมบน Ubuntu Server เพื่อคัดกรองและควบคุมพอร์ตที่อนุญาตให้อุปกรณ์เครือข่ายภายนอกเชื่อมต่อเข้ามายังตัวเครื่องเซิร์ฟเวอร์ได้

> [!CAUTION]
> **ข้อพึงระวังขั้นสูงสุดก่อนเปิดใช้งาน UFW:**
> ก่อนที่จะเปิดใช้งานไฟร์วอลล์ UFW (`ufw enable`) คุณ **ต้องมั่นใจว่าได้สั่งอนุญาตให้พอร์ตเชื่อมต่อ SSH (Port 22 หรือพอร์ตที่กำหนด) เข้ามาได้ก่อนเสมอ** มิฉะนั้น หากเปิดระบบปุ๊บ ระบบจะบล็อกการเข้าถึงทางพอร์ต 22 ทันที และจะทำให้เซสชันรีโมทที่ทำงานอยู่โดนเตะตัดการเชื่อมต่อ และจะไม่สามารถล็อกอินเข้าเครื่องเซิร์ฟเวอร์ได้อีกเลยนับจากนั้น!

### ชุดคำสั่งการควบคุม UFW:

* **`ufw status` / `ufw status verbose`**: ตรวจสอบสถานะการทำงานปัจจุบันของไฟร์วอลล์และกฎระเบียบต่างๆ ที่บันทึกอยู่
  ```bash
  sudo ufw status
  ```
  *(หากสถานะขึ้นว่า `Status: inactive` แปลว่าไฟร์วอลล์ปิดอยู่และกำลังปล่อยให้ทุกพอร์ตเข้าถึงได้)*

* **`ufw allow`**: สั่งสร้างกฎอนุญาตให้เชื่อมต่อพอร์ตหรือบริการตามที่ระบุเข้ามาได้
  ```bash
  sudo ufw allow 22           # เปิดไฟเขียวให้อุปกรณ์ภายนอกเชื่อมต่อ SSH (พอร์ต 22)
  sudo ufw allow 80/tcp       # เปิดเฉพาะการส่งข้อมูลแบบ TCP ของพอร์ต 80 (HTTP Web Server)
  sudo ufw allow 443          # อนุญาตพอร์ต 443 (HTTPS)
  ```

* **`ufw deny`**: สั่งสร้างกฎปฏิเสธ/ปิดกั้นช่องทางพอร์ตหรือบริการตามที่ระบุ
  ```bash
  sudo ufw deny 3306          # ปิดกั้นพอร์ต 3306 (MySQL) ไม่ให้เครื่องข้างนอกเชื่อมเข้ามาได้
  ```

* **`ufw enable`**: เปิดการทำงานของระบบไฟร์วอลล์ UFW ทั้งระบบทันที (จะเริ่มนำกฎ allow/deny ต่างๆ ที่ตั้งไว้มาคัดกรองสัญญาณเข้าจริง)
  ```bash
  sudo ufw enable
  ```

* **`ufw disable`**: สั่งหยุดการทำงานของไฟร์วอลล์ ปล่อยเสรีทุกการเชื่อมต่อพอร์ต
  ```bash
  sudo ufw disable
  ```

* **`ufw reload`**: สั่งให้อ่านกฎระเบียบทั้งหมดที่เขียนเพิ่มเติมใหม่โดยไม่มีการปิดการทำงานของไฟร์วอลล์
  ```bash
  sudo ufw reload
  ```

* **`ufw status numbered`**: ตรวจกฎข้อบังคับเรียงตามหมายเลขลำดับ (ใช้สำหรับการระบุเพื่อลบกฎย่อย)
  ```bash
  sudo ufw status numbered
  ```

* **`ufw delete <หมายเลขกฎ>`**: สั่งลบกฎที่ตั้งค่าไว้ผ่านหมายเลขลำดับที่เปิดดูจากคำสั่ง ufw status numbered
  ```bash
  sudo ufw delete 3           # สั่งลบกฎข้อที่ 3 ของระบบ
  ```
