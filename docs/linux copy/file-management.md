# คู่มือการจัดการไฟล์และโฟลเดอร์อย่างละเอียด

การควบคุม จัดการ ค้นหา และเคลื่อนย้ายไฟล์/โฟลเดอร์ผ่านระบบ Command Line (CLI) บนระบบปฏิบัติการ Linux

---

## 💡 แนวคิดหลักในการทำงานบนเทอร์มินัล (CLI Concepts)

ก่อนเริ่มต้นใช้งานคำสั่ง ควรทำความเข้าใจแนวคิดพื้นฐานในการสั่งการผ่านบรรทัดคำสั่งดังนี้:

### 1. ทำไมถึงไม่มีคำสั่ง "วาง" (Paste) แยกต่างหาก?
บนหน้าต่างโปรแกรม (GUI) การทำงานจะแบ่งเป็น 2 จังหวะ คือคัดลอก (Copy) แล้วจึงไปกดวาง (Paste) แต่ในระบบเทอร์มินัล คำสั่ง **`cp` (Copy)** และ **`mv` (Move)** จะทำงาน **คัดลอก/ตัด และ "วาง" ลงตำแหน่งปลายทางไปพร้อมกันในคำสั่งเดียว** 

* **โครงสร้างไวยากรณ์คำสั่งทั่วไป:**
  ```bash
  คำสั่ง [ออปชันย่อย] <ไฟล์ต้นทาง> <ตำแหน่งปลายทางเพื่อทำการวาง>
  ```
  *(ระบุโฟลเดอร์ปลายทางหรือชื่อใหม่ที่ต้องการวางต่อท้ายคำสั่งได้ทันที)*

### 2. สัญลักษณ์แทนเส้นทางตำแหน่ง (Paths Symbols)
* **`.` (จุดเดียว):** แทน **โฟลเดอร์ปัจจุบันที่ทำงานอยู่ (Current Directory)**
  * *ตัวอย่าง:* `cp /etc/nginx/nginx.conf .` *(คัดลอกไฟล์จากต้นทางมาวางที่โฟลเดอร์ปัจจุบัน)*
* **`..` (สองจุด):** แทน **โฟลเดอร์ชั้นนอกที่อยู่เหนือขึ้นไป 1 ระดับ (Parent Directory)**
  * *ตัวอย่าง:* `cd ..` *(ถอยหลังออกจากโฟลเดอร์ปัจจุบัน 1 ชั้น)*
* **`~` (ตัวหนอน):** แทน **โฟลเดอร์หลักส่วนตัว (Home Directory)** ของผู้ใช้งาน (เช่น `/home/server/` หรือ `/root/`)
  * *ตัวอย่าง:* `cd ~` *(พากลับมายังโฟลเดอร์หลักของผู้ใช้ทันที)*

### 3. ปุ่มช่วยสะกดคำอัตโนมัติ (Tab Autocomplete)
* พิมพ์ตัวอักษรเริ่มต้นของชื่อไฟล์หรือโฟลเดอร์ย่อย แล้วกดปุ่ม **`Tab` บนคีย์บอร์ด 1 ครั้ง** ระบบจะสะกดคำและแสดงชื่อส่วนที่เหลือให้ทันทีเพื่อลดความผิดพลาดในการพิมพ์
* หากกดแล้วเงียบ ให้กดปุ่ม **`Tab` ติดกัน 2 ครั้ง** เพื่อแสดงรายชื่อไฟล์/โฟลเดอร์ทั้งหมดที่มีชื่อขึ้นต้นด้วยตัวอักษรนั้นๆ

## 🔍 คำสั่งจัดการไฟล์และโฟลเดอร์พื้นฐาน (Basic Commands)

### 1. `pwd` (Print Working Directory)
* **คำอธิบาย:** แสดงพาธตำแหน่งเต็มของไดเรกทอรีปัจจุบันที่ผู้ใช้กำลังเปิดใช้งานอยู่ (Absolute Path)
* **ตัวอย่างการรัน:**
  ```bash
  pwd
  ```
  *ผลลัพธ์ที่จะแสดง:* `/var/www/html/my-project`

### 2. `ls` (List)
* **คำอธิบาย:** แสดงรายชื่อไฟล์และโฟลเดอร์ย่อยในตำแหน่งปัจจุบัน
* **ออปชันย่อยที่สำคัญ:**
  * `-l` : แสดงรายละเอียดแบบแถวยาว (Long format) เช่น สิทธิ์ของไฟล์, ขนาดไฟล์, วันเวลาแก้ไขล่าสุด
  * `-a` : แสดงไฟล์ทั้งหมดรวมถึงไฟล์ซ่อน (Hidden Files) ซึ่งจะมีเครื่องหมายจุดนำหน้า เช่น `.env`, `.gitignore`
  * `-h` : แสดงขนาดพื้นที่ของไฟล์ในรูปแบบที่มนุษย์อ่านเข้าใจง่าย เช่น 1.5K, 4.2M, 1.2G (ใช้คู่กับ `-l`)
  * `-t` : จัดเรียงไฟล์ตามวันเวลาที่มีการแก้ไขล่าสุด (ไฟล์ใหม่สุดอยู่ด้านบน)
  * `-S` : จัดเรียงไฟล์ตามขนาดความจุ (ไฟล์ใหญ่ที่สุดอยู่ด้านบน)
* **ตัวอย่างการรัน:**
  ```bash
  ls -lh           # แสดงรายละเอียดของไฟล์ปกติให้อ่านขนาดพื้นที่เข้าใจง่าย
  ls -la           # แสดงรายละเอียดและไฟล์ซ่อนทั้งหมด
  ls -lhS          # แสดงไฟล์ทั้งหมด เรียงตามขนาดจากใหญ่ไปเล็ก
  ```

### 3. `cd` (Change Directory)
* **คำอธิบาย:** เปลี่ยนตำแหน่งโฟลเดอร์การทำงานไปยังตำแหน่งใหม่ที่ระบุ
* **ตัวอย่างการรัน:**
  ```bash
  cd /var/log      # ย้ายไปยังโฟลเดอร์เก็บ log ระบบ (Absolute Path)
  cd docs/linux    # ย้ายเข้าไปยังโฟลเดอร์ย่อยภายใน (Relative Path)
  cd ~             # กลับไปยังโฟลเดอร์หลักของผู้ใช้งาน (Home Folder)
  cd -             # สลับกลับไปยังตำแหน่งล่าสุดก่อนหน้านี้
  ```

### 4. `mkdir` (Make Directory)
* **คำอธิบาย:** สร้างโฟลเดอร์ใหม่ขึ้นมาในระบบ
* **ออปชันย่อยที่สำคัญ:**
  * `-p` : สร้างโฟลเดอร์ตามโครงสร้างชั้นย่อยทั้งหมดที่ยังไม่มีในขณะนั้นโดยอัตโนมัติ (Parent directories creation)
* **ตัวอย่างการรัน:**
  ```bash
  mkdir new-folder                   # สร้างโฟลเดอร์เปล่าใหม่
  mkdir -p project/src/components    # สร้างโฟลเดอร์ project, src และ components ซ้อนกันเป็นชั้นๆ ในคำสั่งเดียว
  ```

### 5. `cp` (Copy)
* **คำอธิบาย:** คัดลอกไฟล์หรือโฟลเดอร์จากต้นทางไปวางไว้ที่ปลายทาง
* **ออปชันย่อยที่สำคัญ:**
  * `-r` : คัดลอกข้อมูลทั้งโฟลเดอร์ รวมถึงโฟลเดอร์ย่อยและไฟล์ภายในทั้งหมด (Recursive)
  * `-f` : บังคับคัดลอกเขียนทับไฟล์ปลายทางโดยไม่ต้องถามยืนยัน (Force)
  * `-p` : คัดลอกโดยรักษาคุณสมบัติสิทธิ์ของไฟล์และวันเวลาของไฟล์ต้นฉบับไว้ (Preserve)
* **ตัวอย่างการรัน:**
  ```bash
  cp doc.txt /var/backup/                   # คัดลอกไฟล์เดี่ยวไปวางในโฟลเดอร์ปลายทาง
  cp doc.txt /var/backup/new_name.txt       # คัดลอกไฟล์ไปวางพร้อมเปลี่ยนชื่อใหม่
  cp -rp project-folder/ /var/www/html/     # คัดลอกทั้งโฟลเดอร์พร้อมรักษาสิทธิ์ของไฟล์ดั้งเดิม
  ```

### 6. `mv` (Move)
* **คำอธิบาย:** ย้ายไฟล์/โฟลเดอร์ไปยังตำแหน่งใหม่ หรือใช้สำหรับเปลี่ยนชื่อไฟล์เดิม
* **ตัวอย่างการรัน:**
  ```bash
  mv data.csv /var/data/                    # ย้ายไฟล์ไปยังตำแหน่งเป้าหมาย (เสมือนการ Cut & Paste)
  mv old-name.txt new-name.txt              # เปลี่ยนชื่อไฟล์ในตำแหน่งเดิม
  mv project/ /var/www/html/                # ย้ายตำแหน่งของโฟลเดอร์ทั้งโฟลเดอร์
  ```

### 7. `rm` (Remove)
* **คำอธิบาย:** ลบไฟล์หรือโฟลเดอร์ออกจากระบบเซิร์ฟเวอร์อย่างถาวร
* **ออปชันย่อยที่สำคัญ:**
  * `-r` : ลบทั้งโฟลเดอร์ รวมถึงเนื้อหาย่อยภายในทั้งหมด (Recursive)
  * `-f` : บังคับลบข้อมูลทันทีโดยไม่ต้องถามขอยืนยัน (Force)
* **ตัวอย่างการรัน:**
  ```bash
  rm file.txt                               # ลบไฟล์เดี่ยวทั่วไป
  rm -f config.log                          # บังคับลบไฟล์ทิ้งโดยไม่มีการกดยืนยัน
  rm -rf folder_name/                       # ลบทั้งโฟลเดอร์ย่อยและไฟล์ทั้งหมดอย่างรวดเร็วและเด็ดขาด
  ```

:::warning

**ระมัดระวังในการใช้งานคำสั่ง `rm -rf`:**
คำสั่งนี้จะทำการลบข้อมูลอย่างถาวรและรวดเร็วโดยไม่มีการถามตรวจสอบหรือกู้คืนได้ การออกคำสั่งเช่น `rm -rf /` หรือระบุพาธผิดพลาดเพียงจุดเดียว อาจทำให้ระบบปฏิบัติการพังเสียหายทั้งหมดทันที

:::

## 🚀 คำสั่งการทำงานและค้นหาขั้นสูง (Advanced Commands)

### 8. `touch`
* **คำอธิบาย:** สร้างไฟล์เปล่าใหม่ขึ้นมา หรือใช้อัปเดตเวลาการแก้ไขล่าสุดของไฟล์ (Timestamp) โดยไม่ส่งผลกระทบต่อข้อมูลเดิมข้างใน
* **ตัวอย่างการรัน:**
  ```bash
  touch .env.production                     # สร้างไฟล์เก็บค่า Config เปล่าขึ้นมา
  ```

### 9. `find`
* **คำอธิบาย:** ค้นหาตำแหน่งไฟล์หรือโฟลเดอร์ในระบบตามเงื่อนไขที่กำหนด เช่น ชื่อ ขนาด ชนิด หรือเวลาที่อัปเดต
* **ออปชันย่อยที่สำคัญ:**
  * `-name` : ค้นหาไฟล์ตามรูปแบบของชื่อไฟล์
  * `-type` : ระบุประเภทสิ่งที่จะค้นหา (`f` แทนไฟล์ธรรมดา, `d` แทนโฟลเดอร์)
  * `-size` : ระบุขนาดความจุของไฟล์ เช่น `+100M` (ใหญ่กว่า 100MB), `-10k` (เล็กกว่า 10KB)
* **ตัวอย่างการรัน:**
  ```bash
  find . -name "*.conf"                      # ค้นหาไฟล์คอนฟิก (.conf) ทั้งหมดในตำแหน่งปัจจุบัน
  find /var/log -type f -name "*.log"        # ค้นหาไฟล์ธรรมดาที่เป็นไฟล์ .log ภายในโฟลเดอร์ /var/log
  find /var/www -type f -size +50M           # ค้นหาไฟล์ที่มีขนาดใหญ่กว่า 50MB ในไดเรกทอรีเว็บเซิร์ฟเวอร์
  ```

### 10. `grep` (Global Regular Expression Print)
* **คำอธิบาย:** ค้นหาคำสำคัญหรือรูปแบบประโยคที่ต้องการระบุภายในเนื้อหาของไฟล์ข้อมูล
* **ออปชันย่อยที่สำคัญ:**
  * `-r` : ค้นหาทุกไฟล์ภายในโฟลเดอร์ย่อยทั้งหมด (Recursive)
  * `-n` : แสดงหมายเลขบรรทัดที่มีคำค้นนั้นๆ ปรากฏอยู่
  * `-i` : ค้นหาโดยไม่สนใจตัวอักษรพิมพ์เล็กหรือพิมพ์ใหญ่ (Case-insensitive)
  * `-v` : แสดงบรรทัดที่ **ไม่มี** คำค้นที่เรากำหนด (Invert match)
* **ตัวอย่างการรัน:**
  ```bash
  grep "error" syslog                       # ค้นหาคำว่า "error" ในไฟล์ syslog
  grep -rn "DB_PASSWORD" .                   # ค้นหาข้อความรหัสผ่านในไฟล์ของระบบทั้งหมดพร้อมบอกแถวบรรทัด
  grep -vi "info" /var/log/messages         # ค้นหาและเปิดดูข้อความที่ไม่มีคำว่า "info"
  ```

### 11. `du` (Disk Usage)
* **คำอธิบาย:** คำนวณและตรวจสอบปริมาณการใช้งานพื้นที่จัดเก็บข้อมูลของไฟล์และโฟลเดอร์ย่อยแต่ละตัว
* **ออปชันย่อยที่สำคัญ:**
  * `-h` : แสดงผลลัพธ์เป็นขนาดพื้นที่อ่านง่าย (GB, MB, KB)
  * `-s` : สรุปขนาดพื้นที่โดยรวมของเป้าหมายนั้นๆ (ไม่แสดงขนาดของไฟล์ย่อยแยกทีละไฟล์)
* **ตัวอย่างการรัน:**
  ```bash
  du -sh *                                  # ตรวจสอบขนาดความจุของแต่ละโฟลเดอร์และไฟล์ย่อยในตำแหน่งปัจจุบัน
  du -sh /var/log/                          # สรุปขนาดพื้นที่เก็บ log ทั้งหมดของระบบ
  ```

### 12. `tar` (Tape Archive)
* **คำอธิบาย:** รวบรวมกลุ่มไฟล์เข้าด้วยกัน บีบอัด หรือแตกไฟล์บีบอัดสกุลมาตรฐานของ Linux (`.tar.gz` หรือ `.tar`)
* **ออปชันย่อยที่สำคัญ:**
  * `-c` : สร้างไฟล์บีบอัดชิ้นใหม่ (Create)
  * `-x` : แตกไฟล์บีบอัดออก (Extract)
  * `-v` : แสดงรายละเอียดรายชื่อไฟล์ระหว่างการบีบอัดหรือแตกไฟล์ (Verbose)
  * `-z` : ทำการบีบอัดหรือแตกไฟล์โดยใช้ gzip (สำหรับนามสกุล `.tar.gz`)
  * `-f` : ระบุชื่อไฟล์ปลายทางในการเซฟข้อมูล
  * `-C` : กำหนดโฟลเดอร์เป้าหมายในการแตกไฟล์
* **ตัวอย่างการรัน:**
  ```bash
  tar -cvzf backup.tar.gz /var/www/html     # บีบอัดโฟลเดอร์เว็บเซิร์ฟเวอร์ให้กลายเป็นไฟล์ backup.tar.gz
  tar -xvzf backup.tar.gz -C /var/backup/   # แตกไฟล์กู้ข้อมูลออกไปยังโฟลเดอร์ /var/backup/
  ```

### 13. `zip` / `unzip`
* **คำอธิบาย:** บีบอัดและแตกไฟล์รูปแบบไฟล์ `.zip` ซึ่งสามารถเปิดอ่านและนำไปใช้ร่วมกับ Windows ได้ง่าย
* **ตัวอย่างการรัน:**
  ```bash
  zip -r project.zip src/ config/           # บีบอัดโฟลเดอร์ src และ config ให้กลายเป็น project.zip (-r คือรวมโฟลเดอร์ย่อย)
  unzip project.zip -d /tmp/project         # แตกไฟล์ zip ไปวางในโฟลเดอร์ปลายทางที่กำหนด (-d)
  ```

### 14. `scp` (Secure Copy)
* **คำอธิบาย:** การคัดลอกและรับส่งไฟล์อย่างปลอดภัยข้ามเครือข่ายระหว่างเครื่องปัจจุบันกับเครื่องเซิร์ฟเวอร์ปลายทางผ่านโปรโตคอล SSH
* **ตัวอย่างการรัน:**
  ```bash
  # ส่งไฟล์จากเครื่องตนเองไปเก็บไว้ที่โฟลเดอร์เซิร์ฟเวอร์ปลายทาง
  scp my-backup.zip root@103.224.86.15:/var/www/backups/

  # ดึงไฟล์สำรองข้อมูลจากเซิร์ฟเวอร์ปลายทางมาเซฟเก็บไว้ที่เครื่องตนเองปัจจุบัน
  scp root@103.224.86.15:/var/www/backups/database.sql .
  ```

### 15. `rsync` (Remote Sync)
* **คำอธิบาย:** ซิงค์ ซิงโครไนซ์ และคัดลอกไฟล์ระหว่างเครื่องเซิร์ฟเวอร์ที่รวดเร็วกว่า scp มาก เนื่องจากระบบจะเช็คและรับส่งเฉพาะส่วนต่างของไฟล์เท่านั้น (Differential transfer) และรองรับการทำซิงค์กลับทางได้ดี
* **ออปชันย่อยที่สำคัญ:**
  * `-a` : ซิงค์สิทธิ์และข้อมูลเชิงลึกทั้งหมดของไฟล์ดั้งเดิม (Archive mode)
  * `-v` : แสดงความเคลื่อนไหวการรับส่งไฟล์ (Verbose)
  * `-z` : บีบอัดข้อมูลระหว่างการโอนย้ายผ่านเครือข่ายเพื่อความรวดเร็ว (Compress)
  * `--delete` : สั่งให้ลบไฟล์ฝั่งปลายทางทิ้งทันทีหากต้นทางได้ถูกลบไปแล้ว เพื่อให้ข้อมูลตรงกัน 100% (Mirroring)
* **ตัวอย่างการรัน:**
  ```bash
  # ซิงค์ข้อมูลเว็บเซิร์ฟเวอร์ไปเครื่องปลายทางอย่างรวดเร็ว
  rsync -avz /var/www/html/ root@103.224.86.15:/var/www/html/

  # ซิงค์ข้อมูลให้ฝั่งปลายทางมีลักษณะและข้อมูลตรงกันเป็นกระจกเงากับฝั่งต้นทาง
  rsync -avz --delete /var/www/html/ root@103.224.86.15:/var/www/html/
  ```
