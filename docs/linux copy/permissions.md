# คู่มือการจัดการสิทธิ์และเจ้าของไฟล์อย่างละเอียด

ทำความเข้าใจระบบความปลอดภัย สิทธิ์การเข้าถึงไฟล์ (File Permissions) และผู้ดูแลความเป็นเจ้าของ (File Ownership) ในระบบปฏิบัติการ Linux

---

## 🔒 แนวคิดหลักของระบบสิทธิ์ใน Linux (Permissions Concept)

ในระบบปฏิบัติการ Linux ไฟล์และโฟลเดอร์ทุกตัวจะมีการผูกมัดเรื่องความปลอดภัยเพื่อกำหนดขอบเขตสิทธิ์ของผู้ใช้งานแต่ละประเภท โดยระบบจะแบ่งกลุ่มผู้อ้างอิงสิทธิ์และชนิดของสิทธิ์ออกเป็นรูปแบบเฉพาะตัว:

### 1. กลุ่มอ้างอิงผู้ใช้งาน (User Classes)
การระบุผู้ที่มีสิทธิ์ยุ่งเกี่ยวกับไฟล์นั้นๆ จะแบ่งออกเป็น 3 กลุ่มหลัก:
* **`User (u)` / Owner:** เจ้าของไฟล์ (ผู้ที่สร้างไฟล์นั้นขึ้นมา)
* **`Group (g)`:** กลุ่มผู้ใช้ (สมาชิกภายในกลุ่มเครือข่ายที่ระบุไว้)
* **`Other (o)`:** บุคคลทั่วไปคนอื่นๆ ที่เข้าใช้งานเซิร์ฟเวอร์ (ไม่ใช่เจ้าของและไม่ได้อยู่ในกลุ่ม)
* **`All (a)`:** หมายถึงกลุ่มคนข้างต้นทั้งหมดรวมกัน

---

### 2. ประเภทของสิทธิ์การเข้าถึง (Permission Types)
สิทธิ์ที่มีการจัดการกำหนดลงไปในไฟล์หรือโฟลเดอร์จะแบ่งออกเป็น 3 ชนิด:

| รหัสสัญลักษณ์ | ชนิดสิทธิ์ | ความหมายสำหรับ **ไฟล์ (File)** | ความหมายสำหรับ **โฟลเดอร์ (Directory)** | ค่าตัวเลขฐาน |
| :---: | :---: | :--- | :--- | :---: |
| **`r`** | **Read** | สามารถเปิดอ่านดูเนื้อหาภายในไฟล์ได้ | สามารถใช้คำสั่ง `ls` เพื่อดูรายชื่อไฟล์ภายในได้ | **`4`** |
| **`w`** | **Write** | สามารถแก้ไข พิมพ์ข้อมูลลงไฟล์ หรือลบไฟล์ได้ | สามารถสร้าง ลบ หรือย้ายไฟล์ภายในโฟลเดอร์ได้ | **`2`** |
| **`x`** | **Execute** | สามารถรันรหัสคำสั่ง (เช่น สคริปต์, โปรแกรม) | สามารถใช้คำสั่ง `cd` เพื่อเข้าสู่ข้างในโฟลเดอร์ได้ | **`1`** |
| **`-`** | **No Permission** | ไม่มีสิทธิ์การทำงานใดๆ เลย | ไม่มีสิทธิ์การทำงานใดๆ เลย | **`0`** |

---

### 3. การแสดงผลรูปแบบสิทธิ์ (Symbolic Notation)
เมื่อคุณรันคำสั่งแสดงไฟล์แบบละเอียดด้วยคำสั่ง `ls -l` ระบบจะแสดงรหัสสิทธิ์ 10 ตัวแรก เช่น:
```bash
-rwxr-xr--  1 server group 1024 Jun 11 20:00 config.json
drwxr-xr-x  2 server group 4096 Jun 11 20:00 public_html
```

#### การแบ่งส่วนตีความหมายของรหัสสิทธิ์ 10 ตัวแรก:
* **ตัวแรกสุด (ตัวที่ 1):** บอกประเภทของรายการข้อมูล
  * `-` : ไฟล์ธรรมดา (Regular File)
  * `d` : โฟลเดอร์/ไดเรกทอรี (Directory)
  * `l` : ทางลัดคีย์ลิงก์ (Symbolic Link)
* **ตัวที่ 2 - 4 (กลุ่มที่ 1):** สิทธิ์ของ **Owner (เจ้าของไฟล์)**
* **ตัวที่ 5 - 7 (กลุ่มที่ 2):** สิทธิ์ของ **Group (กลุ่มสมาชิก)**
* **ตัวที่ 8 - 10 (กลุ่มที่ 3):** สิทธิ์ของ **Others (บุคคลภายนอก)**

* **ตัวอย่าง:** `-rwxr-xr--`
  * ตัวแรกเป็น `-` = เป็นไฟล์ธรรมดา
  * เจ้าของไฟล์มีสิทธิ์ `rwx` = สามารถอ่าน, เขียน, และสั่งรันไฟล์ได้ครบถ้วน
  * กลุ่มมีสิทธิ์ `r-x` = อ่านและสั่งรันไฟล์ได้ แต่เขียน/แก้ไขไม่ได้ (`-`)
  * บุคคลภายนอกมีสิทธิ์ `r--` = อ่านได้อย่างเดียว ไม่สามารถเขียนหรือรันได้เลย

---

### 4. การนับค่าแบบตัวเลข (Octal/Numeric Notation)
เพื่อให้การสั่งการเปลี่ยนสิทธิ์เป็นไปอย่างรวดเร็ว Linux จึงใช้ระบบเลขฐานแปด (0-7) เข้ามาช่วย โดยการนำค่าตัวเลขฐานของสิทธิ์ที่เปิดมาบวกกันในแต่ละกลุ่มผู้ใช้:

* **การคำนวณ:** นำค่าของสิทธิ์ที่ต้องการมาบวกกัน `Read(4) + Write(2) + Execute(1)`
  * หากต้องการให้ **อ่านและเขียนได้เท่านั้น**: `4 + 2 = 6`
  * หากต้องการให้ **อ่านและสั่งรันได้เท่านั้น**: `4 + 1 = 5`
  * หากต้องการให้ **อ่าน เขียน และสั่งรันได้ครบ**: `4 + 2 + 1 = 7`
  * หาก **ไม่มีสิทธิ์เลย**: `0`

> **วิธีอ่านเลข 3 หลัก เช่น `755`:** หลักซ้าย = Owner, หลักกลาง = Group, หลักขวา = Others
> ดังนั้น `755` = Owner ได้ `7` (rwx), Group ได้ `5` (r-x), Others ได้ `5` (r-x)

#### ตัวเลขยอดนิยมที่ใช้งานบ่อย:

| Octal | Symbolic | ใช้กับอะไร | เหตุผล |
|:---:|:---:|:---|:---|
| `755` | `rwxr-xr-x` | โฟลเดอร์ทั่วไป, สคริปต์ที่ทุกคนรันได้ | เจ้าของควบคุมเต็ม, คนอื่นเข้าถึงได้แต่แก้ไขโครงสร้างไม่ได้ |
| `644` | `rw-r--r--` | ไฟล์ข้อมูลทั่วไป, HTML, config สาธารณะ | เจ้าของแก้ไขได้, คนอื่นเปิดอ่านได้อย่างเดียว |
| `600` | `rw-------` | ไฟล์ลับ เช่น `.env`, SSH private key | มีเฉพาะเจ้าของเท่านั้นที่เห็นและแก้ไขได้ |
| `700` | `rwx------` | โฟลเดอร์ส่วนตัว | ปิดกั้นทุกคนนอกจากเจ้าของโดยสมบูรณ์ |
| `777` | `rwxrwxrwx` | *(ไม่แนะนำ)* | ทุกคนแก้ไข/ลบได้ทั้งหมด อันตรายมากในระบบ production |

## 🛠️ คำสั่งสำหรับบริหารจัดการสิทธิ์ (Commands)

### 1. `chmod` (Change Mode)
* **คำอธิบาย:** เปลี่ยนสิทธิ์การเข้าถึงไฟล์และโฟลเดอร์ สามารถเลือกกำหนดแบบใช้ตัวเลข (Numeric) หรือสัญลักษณ์ตัวอักษร (Symbolic)

#### ออปชันย่อยที่สำคัญ:
* `-R` : เปลี่ยนแปลงสิทธิ์ของไฟล์และโฟลเดอร์ย่อยทั้งหมดภายในตำแหน่งนั้นแบบสืบทอดลงไป (Recursive)

#### การตั้งค่าแบบใช้ระบบตัวเลข (Numeric Method):
* **ตัวอย่างการรัน:**
  ```bash
  # กำหนดสิทธิ์ให้ไฟล์สคริปต์ทั่วไป (เจ้าของอ่าน/เขียน/รันได้ คนอื่นเปิดอ่าน/รันได้)
  chmod 755 start.sh

  # กำหนดสิทธิ์ให้ไฟล์เก็บค่า Config สำคัญ (เจ้าของอ่านเขียนได้ คนอื่นเปิดอ่านไม่ได้เลย)
  chmod 600 .env

  # ปรับสิทธิ์ให้กับโฟลเดอร์เว็บเซิร์ฟเวอร์ทั้งหมดรวมถึงไฟล์ย่อยภายในให้เป็น 755
  chmod -R 755 /var/www/html/
  ```

#### การตั้งค่าแบบใช้สัญลักษณ์ตัวอักษร (Symbolic Method):
สามารถใช้เครื่องหมายบวก (`+` เพิ่มสิทธิ์), ลบ (`-` ถอนสิทธิ์), หรือเท่ากับ (`=` กำหนดตายตัว) ร่วมกับตัวย่อของกลุ่มผู้ใช้ได้:
* **ตัวอย่างการรัน:**
  ```bash
  # เพิ่มสิทธิ์รัน (Execute) ให้กับเจ้าของไฟล์ (+x)
  chmod u+x run.sh

  # ถอนสิทธิ์เขียนและแก้ไขไฟล์ออกจากกลุ่มและบุคคลภายนอก (g และ o)
  chmod go-w document.txt

  # เพิ่มสิทธิ์เขียนให้กับสมาชิกในกลุ่มเฉพาะ
  chmod g+w database.db

  # กำหนดสิทธิ์ให้ทุกคน (All) สั่งรันไฟล์สคริปต์นี้ได้
  chmod a+x script.py
  ```

### 2. `chown` (Change Owner)
* **คำอธิบาย:** เปลี่ยนสิทธิ์ความเป็นเจ้าของไฟล์หรือกลุ่มของไฟล์/โฟลเดอร์

#### โครงสร้างคำสั่ง:
```bash
chown [ออปชันย่อย] <เจ้าของใหม่>:<กลุ่มใหม่> <เป้าหมาย>
```

#### ออปชันย่อยที่สำคัญ:
* `-R` : เปลี่ยนเจ้าของและกลุ่มของโฟลเดอร์ย่อยรวมถึงไฟล์ทั้งหมดภายในลงไปพร้อมกัน (Recursive)

#### ตัวอย่างการใช้งานจริง:
```bash
# เปลี่ยนเจ้าของไฟล์ config.php เป็นผู้ใช้ที่ชื่อ root
chown root config.php

# เปลี่ยนเจ้าของเป็น root และเปลี่ยนกลุ่มของไฟล์เป็นกลุ่มชื่อ admin
chown root:admin database.sqlite

# เปลี่ยนเจ้าของเว็บไดเรกทอรีทั้งหมดเป็นของระบบ www-data (มักใช้สำหรับการตั้งค่าเว็บเซิร์ฟเวอร์ Nginx/Apache)
chown -R www-data:www-data /var/www/html/
```

---

### 3. `chgrp` (Change Group)
* **คำอธิบาย:** เปลี่ยนสิทธิ์เฉพาะส่วนของ **กลุ่มผู้ใช้งาน (Group Owner)** เท่านั้น (ใช้เมื่อต้องการคงสิทธิ์ผู้ดูแลระบบเดิมไว้ แต่ต้องการย้ายกลุ่มเข้าถึงชิ้นงาน)
* **ตัวอย่างการรัน:**
  ```bash
  # เปลี่ยนกลุ่มผู้ดูแลไฟล์เป็นกลุ่มชื่อ developers
  chgrp developers project-files/

  # เปลี่ยนกลุ่มของโฟลเดอร์ย่อยทั้งหมดภายในแบบ Recursive
  chgrp -R www-data /var/www/uploads/
  ```

---

### 4. `umask` (Default Permission Mask)
* **คำอธิบาย:** ทุกครั้งที่สร้างไฟล์หรือโฟลเดอร์ใหม่ ระบบจะไม่ได้ให้สิทธิ์ตามที่ขอตรงๆ แต่จะนำค่า `umask` มา "หัก" ออกก่อนเสมอ เพื่อเป็นกลไกป้องกันไม่ให้ไฟล์ใหม่มีสิทธิ์กว้างเกินไปโดยอัตโนมัติ

* **ค่า Default ของระบบ:**
  * ไฟล์ใหม่จะถูกสร้างด้วยสิทธิ์สูงสุดที่ `666` (ไม่มี execute โดย default)
  * โฟลเดอร์ใหม่จะถูกสร้างด้วยสิทธิ์สูงสุดที่ `777`

* **วิธีคำนวณ:** `สิทธิ์จริง = สิทธิ์ตั้งต้น - umask`
  * ถ้า `umask` เป็น `022`: ไฟล์ใหม่จะได้ `666 - 022 = 644`, โฟลเดอร์ใหม่จะได้ `777 - 022 = 755`
  * ถ้า `umask` เป็น `027`: ไฟล์ใหม่จะได้ `666 - 027 = 640`, โฟลเดอร์ใหม่จะได้ `777 - 027 = 750`

* **ตัวอย่างการรัน:**
  ```bash
  # ดูค่า umask ปัจจุบัน
  umask
  # ผลลัพธ์: 0022

  # ดูค่า umask แบบ Symbolic
  umask -S
  # ผลลัพธ์: u=rwx,g=rx,o=rx

  # ตั้งค่า umask ใหม่ชั่วคราว (มีผลเฉพาะ session นี้เท่านั้น)
  umask 027

  # ตั้งค่าถาวร: เพิ่มบรรทัดนี้ใน ~/.bashrc หรือ /etc/profile
  echo "umask 027" >> ~/.bashrc
  ```

### 5. Special Permissions (สิทธิ์พิเศษ)
นอกจาก `rwx` แบบปกติ Linux ยังมีสิทธิ์พิเศษอีก 3 ชนิด ซึ่งมักพบในไฟล์ระบบและมีผลต่อความปลอดภัยอย่างมาก:

#### A. SetUID (`s` บน Owner execute) — สำหรับไฟล์
* **ความหมาย:** เมื่อผู้ใช้รันไฟล์นี้ โปรแกรมจะทำงานในนาม **เจ้าของไฟล์** (ไม่ใช่ผู้รัน) ชั่วคราว
* **ตัวอย่างจริง:** คำสั่ง `passwd` มี SetUID เป็น root เพื่อให้ผู้ใช้ทั่วไปเปลี่ยนรหัสผ่านตัวเองได้ (ซึ่งต้องเขียนลง `/etc/shadow` ที่ root เท่านั้นเข้าถึงได้)
* **สังเกตจาก:** ตัว `s` แทนที่ `x` ในส่วน Owner เช่น `-rwsr-xr-x`

```bash
# ตั้งค่า SetUID ด้วย Symbolic
chmod u+s program

# ตั้งค่าด้วย Octal (เพิ่มเลข 4 นำหน้า)
chmod 4755 program
```

#### B. SetGID (`s` บน Group execute) — สำหรับไฟล์และโฟลเดอร์
* **ความหมายบนไฟล์:** โปรแกรมจะทำงานในนาม **กลุ่มเจ้าของไฟล์** ชั่วคราว (คล้าย SetUID แต่เป็นระดับกลุ่ม)
* **ความหมายบนโฟลเดอร์ (สำคัญมาก):** ไฟล์ใหม่ทุกไฟล์ที่สร้างในโฟลเดอร์นี้จะ **สืบทอดกลุ่มของโฟลเดอร์** โดยอัตโนมัติ แทนที่จะใช้กลุ่มของผู้สร้าง เหมาะมากสำหรับโฟลเดอร์งานทีม
* **สังเกตจาก:** ตัว `s` แทนที่ `x` ในส่วน Group เช่น `drwxrwsr-x`

```bash
# ตั้งค่า SetGID บนโฟลเดอร์ทีม ให้ไฟล์ใหม่สืบทอดกลุ่ม developers
chmod g+s /shared/team-project/

# ตั้งค่าด้วย Octal (เพิ่มเลข 2 นำหน้า)
chmod 2775 /shared/team-project/
```

#### C. Sticky Bit (`t` บน Other execute) — สำหรับโฟลเดอร์
* **ความหมาย:** ในโฟลเดอร์ที่ทุกคนเขียนได้ (เช่น `/tmp`) Sticky Bit จะป้องกันไม่ให้ผู้ใช้คนอื่นลบหรือเปลี่ยนชื่อไฟล์ที่ตัวเองไม่ได้เป็นเจ้าของได้ **ทุกคนสร้างได้ แต่ลบได้เฉพาะของตัวเองเท่านั้น**
* **สังเกตจาก:** ตัว `t` แทนที่ `x` ในส่วน Others เช่น `drwxrwxrwt`
* **ตัวอย่างจริง:** ไดเรกทอรี `/tmp` ของทุกระบบ Linux ใช้ Sticky Bit เพื่อป้องกันผู้ใช้ลบไฟล์ชั่วคราวของคนอื่น

```bash
# ตั้งค่า Sticky Bit บนโฟลเดอร์ที่ใช้งานร่วมกัน
chmod +t /shared/public-uploads/

# ตั้งค่าด้วย Octal (เพิ่มเลข 1 นำหน้า)
chmod 1777 /shared/public-uploads/

# ตรวจสอบว่า /tmp มี Sticky Bit หรือไม่
ls -ld /tmp
# ผลลัพธ์: drwxrwxrwt 10 root root 4096 Jun 12 00:00 /tmp
```

#### สรุปตัวเลข Special Permissions:

| Octal นำหน้า | ชื่อ | ใช้กับ | ผลลัพธ์ |
|:---:|:---:|:---:|:---|
| `4xxx` | SetUID | ไฟล์ | รันในนามเจ้าของไฟล์ชั่วคราว |
| `2xxx` | SetGID | ไฟล์ / โฟลเดอร์ | รันในนามกลุ่ม / ไฟล์ใหม่สืบทอดกลุ่ม |
| `1xxx` | Sticky Bit | โฟลเดอร์ | ลบได้เฉพาะไฟล์ของตัวเอง |
