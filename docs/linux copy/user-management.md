# คู่มือการจัดการผู้ใช้งานและกลุ่มระบบ (User & Group Management) อย่างละเอียด

ระบบปฏิบัติการ Linux ถูกออกแบบมาบนพื้นฐานของระบบ **Multi-user** (รองรับการทำงานของผู้ใช้งานหลายคนพร้อมกัน) การบริหารจัดการบัญชีผู้ใช้ (Users) กลุ่มผู้ใช้ (Groups) สิทธิ์ผู้ดูแลระบบ (Sudo) และการตรวจสอบความเคลื่อนไหวของเซสชัน จึงมีความสำคัญอย่างยิ่งต่อความปลอดภัยและความมั่นคงของระบบ

---

## 📂 1. โครงสร้างการจัดเก็บข้อมูลบัญชีในระบบ (System Configuration Files)

ข้อมูลผู้ใช้งานและกลุ่มใน Linux จะถูกบันทึกอยู่ในไฟล์ข้อความธรรมดา (Plain text) 3 ไฟล์หลักภายใต้ไดเรกทอรี `/etc/` ซึ่งเราจำเป็นต้องเข้าใจโครงสร้างภายในเพื่อการจัดการที่ถูกต้อง:

### A. ไฟล์ `/etc/passwd` (บัญชีรายชื่อผู้ใช้)
เก็บรายชื่อบัญชีผู้ใช้งานระบบทั้งหมด (ทั้งมนุษย์และระบบ/เซอร์วิส) ไฟล์นี้เปิดให้อ่านได้ทั่วไป (Read-only สำหรับผู้ใช้ทั่วไป) โครงสร้างข้อมูลในแต่ละบรรทัดแบ่งออกเป็น **7 ส่วน** คั่นด้วยเครื่องหมายโคลอน (`:`):

```text
server:x:1001:1001:Server Admin,,,:/home/server:/bin/bash
```

* **คำอธิบายทั้ง 7 ส่วน:**
  1. `server` : **Username** ชื่อที่ใช้ในการเข้าสู่ระบบ
  2. `x` : **Password placeholder** (ปัจจุบันย้ายไปเก็บแบบเข้ารหัสในไฟล์ `/etc/shadow`)
  3. `1001` : **UID (User ID)** หมายเลขประจำตัวผู้ใช้ (ค่า `0` คือ root, `1-999` คือระบบ/เซอร์วิส, `1000+` คือผู้ใช้ทั่วไป)
  4. `1001` : **GID (Group ID)** หมายเลขประจำตัวกลุ่มหลัก (Primary Group)
  5. `Server Admin,,,` : **Comment / GECOS** ข้อมูลเพิ่มเติม เช่น ชื่อเต็ม เบอร์โทรศัพท์
  6. `/home/server` : **Home Directory** โฟลเดอร์ส่วนตัวหลักของผู้ใช้งาน
  7. `/bin/bash` : **Login Shell** โปรแกรมเชลล์ที่จะถูกเรียกใช้งานเมื่อล็อกอิน (หากเป็นเซอร์วิสมักกำหนดเป็น `/usr/sbin/nologin`)

### B. ไฟล์ `/etc/shadow` (ที่เก็บรหัสผ่านเข้ารหัสลับ)
เก็บรหัสผ่านจริงที่ผ่านการแฮช (Hashed password) และนโยบายวันหมดอายุของรหัสผ่าน **มีเพียงสิทธิ์ root เท่านั้นที่เปิดอ่านได้** โครงสร้างประกอบด้วยข้อมูลสำคัญ เช่น ชื่อผู้ใช้, รหัสผ่านที่เข้ารหัส, วันที่เปลี่ยนรหัสผ่านล่าสุด, และจำนวนวันก่อนรหัสผ่านหมดอายุ

### C. ไฟล์ `/etc/group` (ข้อมูลระบบกลุ่มผู้ใช้)
เก็บรายชื่อกลุ่มทั้งหมดในระบบและสมาชิกที่สังกัดอยู่ในกลุ่มนั้นๆ แบ่งออกเป็น **4 ส่วน**:
```text
sudo:x:27:server,adminuser
```
*(ชื่อกลุ่ม: `sudo` | รหัสผ่านกลุ่ม: `x` | GID: `27` | สมาชิกในกลุ่ม: `server` และ `adminuser`)*

## 👤 2. ชุดคำสั่งบริหารจัดการบัญชีผู้ใช้งาน (User Commands)

### A. การสร้างผู้ใช้ใหม่: `adduser` vs `useradd`
ในระบบตระกูล Ubuntu/Debian มีคำสั่งสร้างผู้ใช้ 2 รูปแบบ ซึ่งทำงานแตกต่างกันอย่างสิ้นเชิง:

* **คำสั่ง `adduser` (แนะนำสำหรับผู้ใช้ทั่วไป):** เป็น Interactive Script ระดับสูงที่จะนำทางผู้ใช้ไปทีละขั้นตอน โดยระบบจะสร้าง Home Directory, คัดลอกไฟล์คอนฟิกพื้นฐาน (`/etc/skel`), และบังคับให้ตั้งรหัสผ่านทันที
  ```bash
  sudo adduser john
  ```

* **คำสั่ง `useradd` (ระดับล่าง - Low-level):** เป็นคำสั่งดั้งเดิม หากรันเปล่าๆ ระบบจะไม่สร้าง Home Directory และไม่มีรหัสผ่านให้ มักใช้ในการเขียนสคริปต์อัตโนมัติ (Automation)
  
  **ตารางออปชันที่สำคัญของ `useradd`:**
  * `-m` : สั่งให้สร้างโฟลเดอร์ Home Directory ให้ผู้ใช้ใหม่
  * `-s <shell>` : กำหนด Login Shell เริ่มต้น (เช่น `/bin/bash`)
  * `-g <group>` : กำหนดกลุ่มหลัก (Primary Group) โดยระบุเป็นชื่อหรือ GID
  * `-G <groups>` : กำหนดกลุ่มเสริมอื่นๆ (Secondary Groups) คั่นด้วยเครื่องหมายจุลภาค `,`
  * `-c <comment>` : เพิ่มข้อมูลรายละเอียดผู้ใช้ (GECOS)

  **ตัวอย่างการใช้งานจริง:**
  ```bash
  sudo useradd -m -s /bin/bash -g developers -G sudo,www-data bob
  ```

---

### B. การจัดการรหัสผ่าน: คำสั่ง `passwd`
ใช้ในการกำหนด แก้ไข หรือจัดการนโยบายความปลอดภัยของรหัสผ่านผู้ใช้งาน

* **รูปแบบการใช้งานและออปชันย่อย:**
  ```bash
  # เปลี่ยนรหัสผ่านของตนเอง (ไม่ต้องใช้ sudo)
  passwd
  
  # เปลี่ยนรหัสผ่านของผู้ใช้ชื่อ john (ต้องใช้สิทธิ์ sudo)
  sudo passwd john
  
  # [-l] ล็อกบัญชีผู้ใช้ชั่วคราว (Lock) ไม่ให้ล็อกอินเข้าสู่ระบบได้
  sudo passwd -l john
  
  # [-u] ปลดล็อกบัญชีผู้ใช้ (Unlock) ให้กลับมาใช้งานได้ตามปกติ
  sudo passwd -u john
  
  # [-e] บังคับให้รหัสผ่านหมดอายุทันที (Expire) ผู้ใช้ต้องเปลี่ยนรหัสผ่านใหม่ในการล็อกอินครั้งถัดไป
  sudo passwd -e john
  
  # [-d] ลบรหัสผ่านของผู้ใช้ออก ทำให้สามารถล็อกอินได้โดยไม่ต้องใส่รหัสผ่าน (อันตราย!)
  sudo passwd -d john
  ```

---

### C. การแก้ไขข้อมูลผู้ใช้งาน: คำสั่ง `usermod`
ใช้ปรับแต่งค่าและคุณสมบัติต่างๆ ของบัญชีผู้ใช้ที่มีอยู่ในระบบ

* **ออปชันที่สำคัญ:**
  * `-l <ชื่อใหม่>` : เปลี่ยนชื่อ Username (ผู้ใช้ต้องไม่ได้กำลังล็อกอินอยู่)
  * `-g <กลุ่ม>` : เปลี่ยนแปลงกลุ่มหลัก (Primary Group)
  * `-G <กลุ่ม>` : ระบุกลุ่มเสริม (Secondary Group)
  * `-a` : เพิ่มกลุ่มเสริมเข้าไปร่วมกับกลุ่มเดิม (Append) **ต้องใช้คู่กับ `-G` เสมอ**
  * `-d <พาธใหม่>` : เปลี่ยนแปลงตำแหน่ง Home Directory
  * `-m` : ย้ายข้อมูลจากโฟลเดอร์ Home เดิมไปที่ใหม่ (ต้องใช้คู่กับ `-d`)
  * `-s <เชลล์ใหม่>` : เปลี่ยนโปรแกรม Login Shell

* **ตัวอย่างการใช้งานจริง:**
  ```bash
  # เปลี่ยนชื่อผู้ใช้จาก bob เป็น robert
  sudo usermod -l robert bob
  
  # ย้ายโฟลเดอร์ Home ของ robert ไปที่พาธใหม่พร้อมโอนย้ายข้อมูลเดิม
  sudo usermod -m -d /home/robert_new robert
  
  # เปลี่ยน Login Shell ของ john ให้เป็น zsh
  sudo usermod -s /bin/zsh john
  ```
:::warning 

ข้อควรระวังขั้นสูงในการใช้ `-G` <br/>
หากคุณใช้คำสั่ง `usermod -G developers john` โดยไม่มีออปชัน **`-a`** ระบบจะ **ตัดสิทธิ์ผู้ใช้ john ออกจากกลุ่มเสริมเดิมทั้งหมด** (รวมถึงกลุ่ม `sudo`) แล้วให้สิทธิ์กลุ่ม `developers` เพียงกลุ่มเดียว 

**กฎเหล็ก:** ให้ใช้รูปแบบ **`-aG`** ทุกครั้งเพื่อเพิ่มกลุ่มเสริมใหม่โดยไม่กระทบสิทธิ์เดิม:
```bash
sudo usermod -aG sudo john
```
:::


---

### D. การลบบัญชีผู้ใช้งาน: คำสั่ง `userdel`
ใช้ลบบัญชีผู้ใช้งานออกจากระบบปฏิบัติการ

* **ออปชันที่สำคัญ:**
  * `-r` : ลบไดเรกทอรี Home และกล่องจดหมาย (Mail Spool) ของผู้ใช้คนนั้นออกจากดิสก์อย่างถาวร
  * `-f` : บังคับลบ (Force) แม้ว่าผู้ใช้นั้นจะยังคงล็อกอินอยู่ในระบบอยู่ก็ตาม

* **ตัวอย่างการใช้งานจริง:**
  ```bash
  # ลบชื่อบัญชีอย่างเดียว แต่คงโฟลเดอร์ข้อมูลใน Home ไว้เพื่อตรวจสอบ
  sudo userdel john
  
  # ลบบัญชีพร้อมลบข้อมูลส่วนตัวทั้งหมดออกจากระบบทันที
  sudo userdel -r john
  ```

## 👥 3. ชุดคำสั่งบริหารจัดการกลุ่มผู้ใช้งาน (Group Commands)

การจัดกลุ่มมีไว้เพื่อความสะดวกในการจัดการสิทธิ์เข้าถึงไฟล์และไดเรกทอรี (File Permissions) ให้แก่ผู้ใช้หลายคนพร้อมกัน

* **`groupadd` (สร้างกลุ่มใหม่):**
  ```bash
  sudo groupadd developers
  # ออปชัน -g <GID> สามารถใช้กำหนดหมายเลขกลุ่มที่ต้องการได้ เช่น sudo groupadd -g 2000 developers
  ```
* **`groupmod` (แก้ไขกลุ่ม):** ใช้เปลี่ยนชื่อหรือหมายเลข GID ของกลุ่ม
  ```bash
  # เปลี่ยนชื่อกลุ่มจาก developers เป็น dev-team [-n]
  sudo groupmod -n dev-team developers
  
  # เปลี่ยนหมายเลข GID ของกลุ่ม dev-team เป็น 2500 [-g]
  sudo groupmod -g 2500 dev-team
  ```
* **`groupdel` (ลบกลุ่ม):** ลบกลุ่มออกจากระบบ (ผู้ใช้ที่เคยอยู่ในกลุ่มนี้จะไม่ถูกลบ แต่จะหมดสถานะสมาชิกของกลุ่มนี้อัตโนมัติ)
  ```bash
  # ลบกลุ่ม dev-team (จะลบไม่ได้หากยังมีผู้ใช้ที่ใช้กลุ่มนี้เป็น Primary Group)
  sudo groupdel dev-team

  # [-f] บังคับลบกลุ่มแม้ว่าจะยังมีผู้ใช้อยู่ใน Primary Group
  sudo groupdel -f dev-team
  ```

## 🛡️ 4. สิทธิ์ผู้ดูแลระบบและการจัดการความปลอดภัยด้วย `sudo`

ผู้ดูแลระบบสูงสุด (**`root`**) มีอำนาจเด็ดขาดในการแก้ไขและลบทุกอย่างในระบบ การใช้งานบัญชี root โดยตรงจึงมีความเสี่ยงสูง Linux จึงแนะนำให้ใช้บัญชีธรรมดา แล้วใช้คำสั่ง **`sudo` (SuperUser DO)** เพื่อขอใช้สิทธิ์ root ชั่วคราวเฉพาะงานที่จำเป็น

### A. การให้สิทธิ์ Sudo แก่ผู้ใช้ทั่วไป
วิธีที่ง่ายและเป็นมาตรฐานที่สุดใน Ubuntu/Debian คือการเพิ่มผู้ใช้คนนั้นเข้าไปในกลุ่มเสริมที่ชื่อว่า `sudo`:
```bash
sudo usermod -aG sudo john
```
*(เมื่อ john ล็อกอินเข้าสู่ระบบใหม่ เขาจะสามารถใช้สิทธิ์แอดมินได้โดยการพิมพ์ `sudo` นำหน้าคำสั่ง)*

### B. การปรับแต่งกฎอย่างละเอียดผ่านคำสั่ง `visudo`
หากต้องการจำกัดสิทธิ์ให้ผู้ใช้บางคนรันได้เฉพาะบางคำสั่ง เราต้องแก้ไขไฟล์คอนฟิก `/etc/sudoers`

:::danger 
ข้อห้ามเด็ดขาดเพื่อความปลอดภัย <br/>
**ห้ามใช้ Text Editor ทั่วไป เช่น `nano` หรือ `vi` เปิดแก้ไขไฟล์ `/etc/sudoers` โดยตรงเด็ดขาด!** เพราะหากเกิดข้อผิดพลาดทางไวยากรณ์ (Syntax Error) แม้เพียงตัวอักษรเดียว ระบบจะล็อกสิทธิ์ sudo ทั้งเซิร์ฟเวอร์ทันที ทำให้ไม่สามารถแก้ไขกลับคืนได้

**วิธีที่ถูกต้อง:** ให้ใช้คำสั่งเฉพาะทางนี้เท่านั้น:
```bash
sudo visudo
```
*คำสั่งนี้จะเปิดไฟล์ขึ้นมาแก้ไขด้วย Editor ปลอดภัย และจะทำการตรวจสอบความถูกต้องของไวยากรณ์ (Syntax Check) ก่อนบันทึกไฟล์จริง หากพิมพ์ผิด ระบบจะแจ้งเตือนและปฏิเสธการบันทึกเพื่อป้องกันระบบล่ม*
:::

**ตัวอย่างกฎใน `/etc/sudoers`:**
```text
# รูปแบบ: <ผู้ใช้> <hostname>=(<รันในนาม>) <คำสั่งที่อนุญาต>

# อนุญาตให้ john รันได้ทุกคำสั่งในฐานะ root (เทียบเท่าสมาชิกกลุ่ม sudo)
john ALL=(ALL:ALL) ALL

# อนุญาตให้ deploy รันเฉพาะคำสั่ง restart nginx เท่านั้น โดยไม่ต้องใส่รหัสผ่าน
deploy ALL=(ALL) NOPASSWD: /bin/systemctl restart nginx

# อนุญาตให้ทุกคนในกลุ่ม developers รันคำสั่ง apt เท่านั้น
%developers ALL=(ALL) /usr/bin/apt
```

## 👁️ 5. การตรวจสอบรายชื่อเซสชันและสถานะผู้ใช้งาน (User Sessions Checking)

เมื่อต้องการตรวจสอบว่าปัจจุบันมีใครล็อกอินเข้ามาใช้งานเซิร์ฟเวอร์ของเราบ้าง หรือต้องการตรวจสอบสิทธิ์ปัจจุบัน สามารถใช้ชุดคำสั่งต่อไปนี้:

### A. คำสั่ง `whoami`
* **คำอธิบาย:** แสดงชื่อบัญชีผู้ใช้งานปัจจุบันที่กำลังรันคำสั่งบนเทอร์มินัลอยู่
* **ตัวอย่างการรัน:**
  ```bash
  whoami
  # ผลลัพธ์: john
  ```

### B. คำสั่ง `id`
* **คำอธิบาย:** แสดงรายละเอียด UID, GID หลัก และกลุ่มเสริมทั้งหมดที่บัญชีนั้นสังกัดอยู่
* **ตัวอย่างการรัน:**
  ```bash
  id john
  # ผลลัพธ์: uid=1001(john) gid=1001(john) groups=1001(john),27(sudo),1002(developers)
  ```

### C. คำสั่ง `who`
* **คำอธิบาย:** แสดงรายชื่อผู้ใช้ที่กำลังล็อกอินใช้งานเครื่องเซิร์ฟเวอร์อยู่ในปัจจุบัน พร้อมประเภทเซสชัน (เช่น `tty` สำหรับหน้าจอจริง หรือ `pts/0` สำหรับการ SSH รีโมทเข้ามา) และเวลาที่ล็อกอิน
* **ตัวอย่างการรัน:**
  ```bash
  who
  # ผลลัพธ์: john     pts/0        2026-06-12 09:30 (192.168.1.50)
  ```

### D. คำสั่ง `w`
* **คำอธิบาย:** เป็นคำสั่งขั้นสูงที่ขยายขีดความสามารถจาก `who` โดยจะแสดงสถานะของระบบ (Uptime, Load Average) รายชื่อผู้ใช้ และ**แสดงรายละเอียดว่าผู้ใช้แต่ละคนกำลังรันคำสั่งอะไรอยู่ ณ วินาทีนั้น (WHAT)**
* **ตัวอย่างการรัน:**
  ```bash
  w
  ```

### E. คำสั่ง `last`
* **คำอธิบาย:** ดึงข้อมูลจากไฟล์ประวัติ `/var/log/wtmp` เพื่อแสดงรายการเข้าและออกจากระบบ (Login/Logout) ย้อนหลังทั้งหมด รวมถึงบันทึกการรีบูตเครื่องเซิร์ฟเวอร์ด้วย
* **ตัวอย่างการรัน:**
  ```bash
  last
  # ผลลัพธ์จะแสดง: ชื่อผู้ใช้, ช่องทางเชื่อมต่อ, IP ต้นทาง, วันเวลา และระยะเวลาที่ใช้งาน
  ```

### F. ความแตกต่างของการสลับผู้ใช้งาน: `su` vs `su -`
เมื่อต้องการสลับสิทธิ์ (Switch User) ไปเป็นผู้ใช้อื่น หรือ root:

* **คำสั่ง `su john`:** สลับไปเป็นผู้ใช้ john แต่**ยังคงรักษาสภาพแวดล้อม (Environment Variables)** และตำแหน่งไดเรกทอรีเดิมของผู้ใช้คนก่อนหน้าไว้
  ```bash
  su john
  # ยังอยู่ที่ไดเรกทอรีเดิม และ $PATH, $HOME ยังเป็นของผู้ใช้เดิม
  ```

* **คำสั่ง `su - john` (แนะนำ):** สลับสิทธิ์แบบ Login ใหม่ทั้งหมด (Login Shell) ระบบจะทำการรีเซ็ตตัวแปรสภาพแวดล้อม และย้ายตำแหน่งพาธไปยัง Home Directory ของ john ทันที
  ```bash
  su - john
  # $HOME จะกลายเป็น /home/john และ Environment จะถูก reset ใหม่ทั้งหมด
  ```

* **คำสั่งสลับเป็นสิทธิ์ root โดยสมบูรณ์** มีสองวิธี:
  ```bash
  # วิธีที่ 1: ใช้ sudo เพื่อเปิด root shell (แนะนำบน Ubuntu/Debian)
  sudo -i

  # วิธีที่ 2: สลับไปเป็น root โดยตรง (ต้องรู้รหัสผ่าน root)
  su - root
  # หรือย่อเพียง
  su -
  ```

### G. การตรวจสอบรายชื่อผู้ใช้ที่มีในระบบ

ต่างจากคำสั่ง `who` หรือ `w` ที่ดูเฉพาะผู้ที่กำลัง **ล็อกอินอยู่** — คำสั่งเหล่านี้ใช้ดู **บัญชีผู้ใช้ทั้งหมดที่ถูกสร้างไว้ในระบบ** (ไม่ว่าจะออนไลน์อยู่หรือไม่ก็ตาม)

* **อ่านจากไฟล์ `/etc/passwd` โดยตรง:**
  ```bash
  # แสดงข้อมูลดิบทุกบัญชีในระบบ (ทั้ง user จริงและ system user)
  cat /etc/passwd

  # กรองเฉพาะชื่อ Username คอลัมน์เดียว (ฟิลด์ที่ 1 คั่นด้วย :)
  cut -d: -f1 /etc/passwd
  ```

* **กรองเฉพาะ User ที่เป็นคนจริง (UID ≥ 1000):**
  ```bash
  # แสดงเฉพาะบัญชีที่สร้างโดยผู้ดูแลระบบ ไม่รวม system/service accounts
  awk -F: '$3 >= 1000 && $3 != 65534 {print $1, $3, $6}' /etc/passwd
  # ผลลัพธ์ตัวอย่าง: john 1001 /home/john
  ```

* **ใช้คำสั่ง `getent` (แนะนำในระบบ enterprise):**
  ```bash
  # getent รองรับทั้ง local users และ users จาก LDAP/Active Directory
  getent passwd

  # กรองเฉพาะชื่อ user
  getent passwd | cut -d: -f1
  ```

* **ดูรายชื่อกลุ่มทั้งหมดในระบบ:**
  ```bash
  # อ่านจากไฟล์โดยตรง
  cat /etc/group

  # แสดงเฉพาะชื่อกลุ่ม
  cut -d: -f1 /etc/group
  ```
