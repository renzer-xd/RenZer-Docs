# คู่มือการตรวจสอบทรัพยากรและโปรเซสระบบอย่างละเอียด

แนวทางการเช็คขนาดความจุฮาร์ดดิสก์ หน่วยความจำ การทำงานของ CPU โปรเซสของระบบ และคำสั่งจัดการยุติโปรเซสที่เกิดปัญหา (Kill Processes)

---

## 💾 1. การตรวจสอบพื้นที่เก็บข้อมูล (Disk Space)

การตรวจสอบว่าเซิร์ฟเวอร์มีพื้นที่จัดเก็บข้อมูลเหลือมากน้อยเพียงใด:

### คำสั่ง `df` (Disk Free)
* **`df -h`**: แสดงพื้นที่เก็บข้อมูลทั้งหมดและการใช้งานในรูปแบบหน่วยพื้นที่อ่านง่าย (GB, MB)
  ```bash
  df -h
  ```
* **`df -i` (Inodes Checking)**: ตรวจสอบจำนวน **Inode** ที่เหลืออยู่
  ```bash
  df -i
  ```
  > [!IMPORTANT]
  > **Inode คืออะไร?**
  > ระบบ Linux จะมีข้อจำกัดเรื่องจำนวนไฟล์สูงสุดที่สามารถบันทึกได้ต่อหนึ่งไดรฟ์ (เรียกว่าจำนวน Inode) หากเซิร์ฟเวอร์ของคุณมีจำนวนไฟล์เล็กๆ มากเกินไป (เช่น ไฟล์ session, cache หรือ temp files) ค่า Inode อาจจะเต็ม 100% ซึ่งทำให้เซิร์ฟเวอร์เกิดอาการ **"ไม่สามารถเขียนข้อมูลใหม่ลงบนดิสก์ได้ (No space left on device)"** ทั้งๆ ที่เมื่อใช้คำสั่ง `df -h` แล้วพื้นที่ยังเหลือว่างอยู่ก็ตาม

---

## 🧠 2. การตรวจสอบหน่วยความจำ (RAM Usage)

การวิเคราะห์ปริมาณการใช้งานแรมของเซิร์ฟเวอร์:

### คำสั่ง `free`
* **`free -h`**: แสดงปริมาณการใช้งานแรมทั้งหมด แรมที่ใช้ แรมที่ว่าง และแรมที่ใช้ทำระบบแคชในหน่วยที่มนุษย์อ่านเข้าใจ
  ```bash
  free -h
  ```
* **`free -m`**: แสดงผลหน่วยพื้นที่จำกัดเฉพาะเจาะจงเป็น Megabyte (MB)
  ```bash
  free -m
  ```

#### การตีความหมายของตารางแรม (RAM Column Info):
* **total:** แรมจริงทั้งหมดที่มีอยู่ในเครื่องเซิร์ฟเวอร์
* **used:** แรมที่โปรเซสต่างๆ กำลังเรียกใช้งานอยู่ในขณะนี้
* **free:** แรมที่ว่างเปล่าร้อยเปอร์เซ็นต์และยังไม่มีโปรแกรมใดเรียกใช้
* **buff/cache:** แรมที่ระบบปฏิบัติการนำไปใช้ชั่วคราวในการพักไฟล์ที่อ่านบ่อยเพื่อทำให้การโหลดข้อมูลเร็วขึ้น (เป็นแรมที่ระบบจะดึงคืนมาใช้งานทันทีเมื่อโปรแกรมทั่วไปต้องการใช้แรม)
* **available:** **ค่าสำคัญที่สุด** คือปริมาณแรมจริงที่ระบบสามารถดึงมาแบ่งปันเพื่อใช้เปิดแอปพลิเคชันหรือโปรเซสใหม่ได้

---

## 📊 3. การเฝ้าตรวจสอบทรัพยากรเรียลไทม์ (Resource Monitoring)

เครื่องมือตรวจสอบการใช้งาน CPU, RAM, และโปรเซสของระบบแบบสดอัปเดตตลอดเวลา:

### A. คำสั่ง `top` (Table of Processes)
* **คำอธิบาย:** โปรแกรมเช็คทรัพยากรพื้นฐานติดเครื่องมาแต่แรก
* **ตัวอย่างการรัน:**
  ```bash
  top
  ```
* **คีย์ลัดสั่งการภายในโปรแกรม:**
  * กดปุ่ม **`M`** (Shift + m) : จัดเรียงโปรเซสตามปริมาณการใช้งานหน่วยความจำ (RAM)
  * กดปุ่ม **`P`** (Shift + p) : จัดเรียงโปรเซสตามปริมาณการดึงกำลังของหน่วยประมวลผล (CPU)
  * กดปุ่ม **`q`** : เพื่อออกจากหน้าจอโปรแกรมกลับมาเทอร์มินัลปกติ

### B. คำสั่ง `htop` (Highly Interactive top)
* **คำอธิบาย:** เครื่องมือแสดงข้อมูลการใช้ทรัพยากรเครื่องแบบกราฟิกสีสันสวยงาม อ่านง่ายกว่า top มาก (หากยังไม่มีในเครื่อง สามารถติดตั้งได้ด้วย `sudo apt install htop`)
* **ตัวอย่างการรัน:**
  ```bash
  htop
  ```
* **สไตล์การแสดงผล:** แสดงความจุและภาระการทำงานของ CPU แยกแต่ละคอร์ พร้อมบอกแถบสีวิเคราะห์แรม ค้นหาคำ และสามารถกดลบโปรเซสได้ทันที
* **การออกจากโปรแกรม:** กดปุ่ม **`q`** หรือปุ่ม **`F10`**

## ⚙️ 4. การจัดการโปรเซสและการปิดโปรแกรม (Processes Management)

โปรเซส (Process) คือโปรแกรมหรือบริการที่ทำงานอยู่เบื้องหลังในหน่วยความจำ ทุกโปรเซสจะมีรหัสตัวเลขประจำตัวเรียกว่า **PID (Process ID)**

### A. การเรียกดูรายการโปรเซส
* **`ps aux`**: ค้นหาและแสดงผลรายการโปรเซสทั้งหมดที่กำลังรันอยู่ในระบบ
  ```bash
  ps aux
  ps aux | grep nginx    # กรองดูเฉพาะโปรเซสที่เกี่ยวกับ nginx
  ```
  *รายละเอียดคอลัมน์สำคัญ:*
  * `USER` : ผู้ใช้งานที่สั่งรันโปรเซสนั้น
  * `PID` : รหัสประจำตัวโปรเซส (ใช้สำหรับสั่งระงับการทำงาน)
  * `%CPU` / `%MEM` : เปอร์เซ็นต์การใช้งานหน่วยประมวลผลและแรม

---

### B. การระงับการทำงานและการปิดโปรเซส (Kill Commands)

หากแอปพลิเคชันของคุณเกิดอาการค้าง (Hang) หรือไม่ยอมตอบสนอง สามารถสั่งยุติการทำงานได้โดยระบุ PID:

* **`kill <PID>` (SIGTERM 15):** สั่งยุติโปรเซสแบบปกติอย่างประนีประนอม เพื่อปล่อยให้โปรแกรมได้เซฟข้อมูลหรือปิดการเชื่อมต่อฐานข้อมูลให้เรียบร้อยก่อนปิด
  ```bash
  kill 1234
  ```

* **`kill -9 <PID>` (SIGKILL 9):** สั่งปิดโปรเซสทันทีอย่างเด็ดขาดและรุนแรง (Force Kill) ระบบปฏิบัติการจะตัดขาดกำลังและปิดแอปพลิเคชันทันทีโดยไม่ฟังเสียงตอบรับของโปรแกรม *มักใช้เมื่อโปรแกรมเกิดอาการค้างหนักและไม่สามารถตอบสนองต่อคำสั่งปิดแบบปกติได้*
  ```bash
  kill -9 1234
  ```

* **`killall <ชื่อโปรแกรม>`:** สั่งปิดทุกโปรเซสที่ทำงานภายใต้ชื่อโปรแกรมนั้นๆ
  ```bash
  killall nginx          # สั่งยุติโปรเซส Nginx ทั้งหมดทุกโปรเซสย่อย
  ```

* **`pkill <ชื่อโปรแกรม>`:** คล้ายกับ killall แต่สามารถระบุคำค้นเป็นชื่อย่อหรือบางส่วนได้
  ```bash
  pkill -9 node          # บังคับลบโปรเซส Node.js ทั้งหมดที่ค้างอยู่
  ```

## 📈 5. คำสั่งเช็คระบบเสริมเพิ่มเติม

* **`uptime`**: เช็คว่าเซิร์ฟเวอร์เปิดใช้งานติดต่อกันมาเป็นระยะเวลานานเท่าใดแล้ว และตรวจสอบค่า **Load Average** (ภาระงานเฉลี่ยย้อนหลัง 1, 5, และ 15 นาทีของ CPU)
  ```bash
  uptime
  ```
* **`lsof` (List Open Files):** ตรวจสอบไฟล์หรือพอร์ตเน็ตเวิร์กที่โปรเซสกำลังเปิดใช้งานอยู่
  ```bash
  lsof -i :80            # ตรวจสอบดูว่ามีใครกำลังเปิดใช้งานพอร์ต 80 (HTTP) อยู่หรือไม่
  ```
* **`uname -a`**: ตรวจเช็คข้อมูลสถาปัตยกรรมของตัวระบบปฏิบัติการและเวอร์ชันของ Linux Kernel
  ```bash
  uname -a
  ```
